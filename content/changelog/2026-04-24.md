## Security

### 2FA now enforced for all staff accounts (RNT-SEC-030)

Agents, agency admins, managing agents, and estate agents must set up an
authenticator app before they can use Klikk. The system blocks login (not
just warns) once the grace period expires. Property owners see a friendly
prompt on first login and can skip it for now — they are not blocked.
Accountants, viewers, and tenants are not affected.

Decision reference: DEC-018.

### Lease builder IDOR fix (RNT-QUAL-052)

An agent could previously create a lease builder session against a lease
belonging to a different agency. This is now blocked — the endpoint checks
the canonical access-control list (the same one used by every other
lease/maintenance/payment endpoint) and returns 403 for unauthorised
attempts.

### Hardcoded test email removed from E-Signing panel (RNT-QUAL-054)

A developer email address (mc@tremly.com) and a dummy phone number were
pre-populated as defaults in the e-signing panel. These have been removed
from production builds. In development, a dev-only stub is still available
via environment variable.

## Compliance

### Legal pages and consent tracking live (OPS-004)

Klikk now has a complete set of SA-specific legal pages on the website:
Privacy Policy, Terms of Service, POPIA statement, PAIA manual, Cookie
Policy, and Acceptable Use Policy. All pages are tagged "pending attorney
review" until OPS-024 is complete.

New agency sign-ups must accept the Terms and Privacy Policy before their
account is created. If the Terms are updated in future, existing users will
be prompted to re-accept on next login. All consent events are recorded with
timestamp, IP address, and document version for POPIA audit purposes.

## Infrastructure

### Automated daily backups to S3 (OPS-003)

A nightly backup job now runs at 02:00 SAST. It dumps the Postgres database,
encrypts it with an age key, and uploads it to S3 (af-south-1). Media files
(including Vault encrypted blobs) are synced separately. Retention: 30 daily
dumps, 12 monthly. If the job fails, a Slack alert fires. In production, a
missing credential is fatal (not a silent skip) to prevent the backup
appearing healthy when it is not. Live restore drill is tracked in OPS-026
pending AWS infrastructure provisioning.

### Task board duplicate guard (OPS-025)

A pre-commit hook and CI check now fail if the same task ID appears in more
than one state folder simultaneously (e.g. both `done/` and `backlog/`). Four
pre-existing stale copies were removed as part of this work.

## Quality

### Push token platform validation (RNT-QUAL-050)

The push token registration endpoint now validates the `platform` field and
returns 400 for any value other than `ios` or `android`. Previously any
string was accepted silently.

### Push token DELETE now returns 204 (RNT-QUAL-055)

The delete-push-token endpoint was returning 200 instead of 204 No Content.
Fixed. An explicit 400 is also returned when the token body parameter is
missing, rather than silently doing nothing.

### Vault33 tests skip cleanly in Rentals environment (RNT-QUAL-053)

Three Vault33 integration tests were causing import errors when running the
Rentals test suite (vault33_client is a separate product not installed here).

## Agent & Tenant App

### Invoice approval panel in maintenance detail (RNT-QUAL-056)

Agents can now approve, reject, or mark invoices as paid directly from the
maintenance request detail screen — no more going to the API. The panel shows
the invoice status chip, total amount, and a PDF download link. Approve goes
through in one click; reject requires a written reason; marking paid asks for
the EFT reference. The activity log refreshes automatically after each action.

### Tenant chat error handling (UX-015)

If a tenant taps the chat FAB or a prompt chip and the server is unreachable
(network offline, session expired), the app now shows a clear error toast
instead of silently doing nothing. A 401/403 redirects to the login screen.

### Branded app icons and splash screens (RNT-QUAL-063)

The agent app no longer shows the generic Android robot icon or a blank splash.
All sizes — iOS App Store icon (1024px), Android adaptive icons (mdpi through
xxxhdpi), and all splash screen densities — are replaced with a Navy/white Klikk
monogram. Note: these are high-quality branded placeholders pending final artwork
from the brand team; App Store / Play Console submission test is deferred to the
next build cycle.

## Security & Compliance

### Landlord banking fields masked in session recordings (RNT-SEC-046)

The representative ID number, branch code, and bank account number on the
Landlord tab are now masked in Clarity session recordings using the same
standard component used across the rest of the platform. Previously these
three fields used a fragile attribute-based approach that did not meet the
post-RNT-SEC-042 standard.

### Audit log now captures actor, IP, and browser for all events (RNT-SEC-039)

Audit events triggered by background signals (e.g. a lease status change fired
by a save) previously had no actor, IP address, or browser attribution — making
POPIA/FICA dispute cross-referencing rely solely on server logs. A new
middleware now captures this context for the duration of every request and
attaches it to any audit event written during that request. A spoofing-negative
test confirms a forged X-Forwarded-For header cannot plant a false IP into the
audit trail. Role-based access to the audit endpoints is also now covered by a
regression test suite (all 9 roles verified).

## Quality & Infrastructure

### AI scrubber no longer over-redacts rand amounts and dates (RNT-QUAL-057)

The AI scrubber was redacting any sequence of 8 or more digits, which meant rent
amounts like R150,000, dates like 20241215, and meter readings like 458790 kWh
were being replaced with [REDACTED] in tenant chat. Bank account numbers are now
only redacted when accompanied by a contextual cue (account, acc, EFT, IBAN,
etc.). Rand amounts, YYYYMMDD dates, and unit-suffixed numbers are explicitly
excluded.

### Duplicate push notifications prevented (RNT-QUAL-061)

Saving a lease, invoice, or maintenance record without changing its status
could previously re-fire a push notification (e.g. a landlord receiving a
second "Rent received" push hours later when an unrelated field was updated).
A pre-save snapshot now guards every push signal so notifications only fire
when the status genuinely transitions.

### Health monitor endpoint open to agents (RNT-QUAL-051)

The maintenance health monitor endpoint was returning 403 to agent users,
matching no other monitor endpoint in the file. Permission corrected to match
the rest — agents and admins can access it; tenants, owners, and suppliers
cannot.

### Full test suite now runs clean (QA-020, QA-021, QA-022)

343 cross-module test failures when running the full test_hub suite are
resolved. Root causes: stale ContentType cache between xdist workers (now
flushed before and after every test), and the test DB being destroyed after
the first TestCase class in a module (fixed by `--reuse-db`). A DRF browsable
API HTML renderer issue that caused JSON parse failures is fixed by a shared
`_JsonAPIClient` that defaults `Accept: application/json`. Regression guard
tests added; `backend/TESTING.md` documents the rules for future test authors.

### Drift-guard parser aligned (RNT-QUAL-058)

The feature-flag drift check script and its Vitest counterpart now share
identical boundary-reset logic, preventing a future edge case where a
slug-less YAML block could cause the test parser to carry a stale slug
forward and produce a false-clean CI result.

### CAPACITOR_SERVER_URL documented for new developers (RNT-QUAL-062)

The agent app `.env.development.example` now includes a commented
`CAPACITOR_SERVER_URL` entry with a dev-only warning. The README has a new
"Live-reload on device" section explaining how to configure it for on-device
hot reload without reading the Capacitor config source directly.

### Pricing v1.1 analysis document (RNT-QUAL-049)

A post-launch pricing review document has been produced at
`docs/strategy/RNT-QUAL-049-pricing-v1.1-review.md`. Key findings: the
Enterprise→Agency tier rename is safe as a display-name-only change (no code
migration); flat-rate pricing outperforms per-unit at current ICP portfolio
sizes (20–100 units); per-unit overage above 100 units is a small revenue
lever worth evaluating. No changes to pricing.yaml — all require MC sign-off
before any copy or config is updated.
