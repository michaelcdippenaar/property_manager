## Security

### 2FA now enforced for all staff accounts (RNT-SEC-030)

Agents, agency admins, managing agents, and estate agents must set up an
authenticator app before they can use Klikk. The system blocks login (not
just warns) once the grace period expires. Property owners see a friendly
prompt on first login and can skip it for now — they are not blocked.
Accountants, viewers, and tenants are not affected.

Decision reference: DEC-018.

### Lease builder IDOR fix (RNT-QUAL-052)

An agent could previously create a lease builder session against a lease
belonging to a different agency. This is now blocked — the endpoint checks
the canonical access-control list (the same one used by every other
lease/maintenance/payment endpoint) and returns 403 for unauthorised
attempts.

### Hardcoded test email removed from E-Signing panel (RNT-QUAL-054)

A developer email address (mc@tremly.com) and a dummy phone number were
pre-populated as defaults in the e-signing panel. These have been removed
from production builds. In development, a dev-only stub is still available
via environment variable.

## Compliance

### Legal pages and consent tracking live (OPS-004)

Klikk now has a complete set of SA-specific legal pages on the website:
Privacy Policy, Terms of Service, POPIA statement, PAIA manual, Cookie
Policy, and Acceptable Use Policy. All pages are tagged "pending attorney
review" until OPS-024 is complete.

New agency sign-ups must accept the Terms and Privacy Policy before their
account is created. If the Terms are updated in future, existing users will
be prompted to re-accept on next login. All consent events are recorded with
timestamp, IP address, and document version for POPIA audit purposes.

## Infrastructure

### Automated daily backups to S3 (OPS-003)

A nightly backup job now runs at 02:00 SAST. It dumps the Postgres database,
encrypts it with an age key, and uploads it to S3 (af-south-1). Media files
(including Vault encrypted blobs) are synced separately. Retention: 30 daily
dumps, 12 monthly. If the job fails, a Slack alert fires. In production, a
missing credential is fatal (not a silent skip) to prevent the backup
appearing healthy when it is not. Live restore drill is tracked in OPS-026
pending AWS infrastructure provisioning.

### Task board duplicate guard (OPS-025)

A pre-commit hook and CI check now fail if the same task ID appears in more
than one state folder simultaneously (e.g. both `done/` and `backlog/`). Four
pre-existing stale copies were removed as part of this work.

## Quality

### Push token platform validation (RNT-QUAL-050)

The push token registration endpoint now validates the `platform` field and
returns 400 for any value other than `ios` or `android`. Previously any
string was accepted silently.

### Push token DELETE now returns 204 (RNT-QUAL-055)

The delete-push-token endpoint was returning 200 instead of 204 No Content.
Fixed. An explicit 400 is also returned when the token body parameter is
missing, rather than silently doing nothing.

### Vault33 tests skip cleanly in Rentals environment (RNT-QUAL-053)

Three Vault33 integration tests were causing import errors when running the
Rentals test suite (vault33_client is a separate product not installed here).
They now skip cleanly with a clear reason message.

## Content

### Pricing copy aligned across all sales materials (GTM-008)

All sales and marketing content now uses the correct pricing tiers (Free,
Pro, Enterprise, Custom) and flat-rate pricing. Per-unit pricing language
("R49/unit", "R39/unit") has been removed. ICP, objections, demo script,
positioning, and competitive matrix were all updated. pricing.yaml is the
single source of truth.

### DEC-reference check narrowed to frontmatter (OPS-027)

The task board integrity script previously scanned entire task files for
DEC-NNN references, causing false positives when prose mentioned a DEC ID
as an example. It now only checks the `depends_on:` frontmatter field.

### Ops skill apex-sender update (OPS-028)

The deployment skill now documents the correct email configuration (apex
sender klikk.co.za, combined SPF with Google Workspace and SES, DKIM on
apex, single DMARC). The superseded mail.klikk.co.za subdomain design has
been removed.
