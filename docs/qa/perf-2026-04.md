## Network profile — SA mobile context

SA mobile networks average 20–50 ms RTT on 4G LTE with throughput around 5–15 Mbps.
3G simulation (150 ms RTT, 1.6 Mbps) is used for Lighthouse budget assertions to reflect rural
and lower-end device conditions common among Klikk's tenant base.

## Tooling installed

### Frontend bundle analyser

`rollup-plugin-visualizer` added to all three frontends. Each CI build emits
`dist/bundle-stats.html` (treemap, gzip sizes) as a retained artefact (30-day TTL).

- `admin/vite.config.ts` — visualizer + `manualChunks` splitting TipTap, pdfjs, Vue core
- `tenant/vite.config.ts` — visualizer + `manualChunks` splitting Vue core
- `website/astro.config.mjs` — visualizer + `manualChunks` splitting Vue islands

### Lighthouse CI

`website/.lighthouserc.json` — 3-run median, simulated 3G mobile. Hard-fail on LCP > 2.5 s.
Workflow: `.github/workflows/lighthouse.yml` — runs on every PR and push to `main`.

### Backend: django-silk (dev profiler)

`django-silk>=5.3.0` added to `requirements.txt`. Enabled in `local.py` only:

- Intercepts `api/v1/leases/*`, `api/v1/esigning/*`, `api/v1/payments/*`
- UI at `http://localhost:8000/silk/`
- Automatically stripped from the test runner (no silk tables in test DB)
- **Not** loaded in `staging.py` or `production.py` — zero prod overhead

### Backend: Sentry APM (staging + production)

Sentry performance tracing is already configured in `base.py`:
```
SENTRY_TRACES_SAMPLE_RATE = 0.05  # 5% of transactions sampled
```
Transaction spans are emitted for Django view processing on all endpoints including
`leases/`, `esigning/`. Raise `SENTRY_TRACES_SAMPLE_RATE` to `1.0` during a targeted
load test to capture full p95 data.

## Baseline measurements (pre-budget enforcement)

Baseline figures not yet captured — CI artefacts from the first post-merge run will
establish the baseline. The following findings are from code inspection:

### Admin SPA — known bundle risks

| Chunk (approximate) | Estimated gz size | Status |
|---------------------|-------------------|--------|
| Vue + Vue Router + Pinia | ~40 KB | Split to `vendor-vue` chunk |
| TipTap suite (starter-kit + extensions) | ~80–120 KB | Split to `vendor-tiptap` chunk |
| pdfjs-dist | ~150–250 KB | Split to `vendor-pdfjs` chunk |
| App code (entry chunk) | ~60–80 KB | Target for 350 KB budget |

The TipTap and pdfjs chunks are loaded lazily (only on lease editor / PDF viewer routes)
and do not contribute to the initial page weight measured by the CI budget check.

### Tenant web — known bundle risks

| Chunk | Estimated gz size | Status |
|-------|-------------------|--------|
| Vue + Vue Router + Pinia | ~40 KB | Split to `vendor-vue` chunk |
| App code | ~30–50 KB | Well within 250 KB budget |

### Marketing website

Astro generates per-page JS islands — only Vue interactive components hydrate on the client.
Static content pages should have near-zero JS on the critical path.
FCP risk: web fonts (Bricolage Grotesque + DM Sans) loaded without `font-display: swap` could
block rendering. **Action:** verify font loading strategy in `website/src/` layout.

---

## Action items for P1 performance wins

1. **Verify `font-display: swap`** on marketing site font imports — prevents render-blocking.
2. **Lazy-load TipTap** in admin SPA lease editor route using dynamic `import()` — currently
   all TipTap extensions are imported at module evaluation time in the editor component.
3. **Add `@sentry/tracing` custom spans** to the most expensive lease generation and PDF export
   views to make p95 visible in Sentry without waiting for sampled transactions.
4. **Run a targeted load test** against staging `leases/` and `esigning/` with k6 or locust to
   capture actual p95 under load before client onboarding.

## CI green criteria

A PR is considered performance-compliant when all of these pass:

- [ ] `lighthouse-website` job: LCP assertion green (no hard-fail on any of 3 runs)
- [ ] `bundle-admin` job: entry chunk <= 350 KB gzipped
- [ ] `bundle-tenant` job: entry chunk <= 250 KB gzipped
- [ ] `backend` job: existing pytest suite still green (silk does not break tests)
