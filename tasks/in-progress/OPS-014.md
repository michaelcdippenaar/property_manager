## Goal
Remove the exposed staging server SSH password from the committed runbook and lock down the server to key-based auth only, eliminating a directly exploitable credential leak.

## Acceptance criteria
- [ ] Inline `# password: pass` comments removed from `deploy/DEVOPS.md` lines 19 and 276
- [ ] Staging server SSH password rotated (server-side action by MC)
- [ ] `PasswordAuthentication no` set in `/etc/ssh/sshd_config` on staging server; `sshd` reloaded — key-based auth only
- [ ] MC's personal SSH public key added to `~/.ssh/authorized_keys` on staging server (the CI deploy key from OPS-012 §5a already satisfies CI access)
- [ ] DEVOPS.md SSH examples updated to show key-based auth (`ssh -i ~/.ssh/klikk_staging mc@102.135.240.222`)

## Files likely touched
- `deploy/DEVOPS.md`
