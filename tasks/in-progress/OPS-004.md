## Goal
Before Klikk accepts a paying customer and their tenants' PI, we need a complete, SA-specific set of legal pages accessible from app + website, and a PAIA manual available on request.

## Acceptance criteria
- [ ] `website/src/pages/legal/privacy.astro` — POPIA-compliant Privacy Policy, SA context, lists info officer contact, lawful bases (s11), retention, DSAR process, cross-border (s72 — we are af-south-1 only)
- [ ] `website/src/pages/legal/terms.astro` — Terms of Service (platform, not tenancy), acceptable use, liability carve-outs, jurisdiction (SA)
- [ ] `website/src/pages/legal/popia.astro` — POPIA statement + Operator/Responsible Party framing
- [ ] `website/src/pages/legal/paia-manual.astro` — PAIA s51 manual with Form 2 template download
- [ ] Sign-up flow in admin + tenant app requires acknowledging current version (timestamped)
- [ ] Version history kept; material changes trigger re-acknowledgement
- [ ] Info Officer named + contactable (POPIA s56)
- [ ] Reviewed against `klikk-legal-POPIA-RHA` skill; defers to that skill for SA citations

## Files likely touched
- `website/src/pages/legal/*.astro` (4 new pages)
- `backend/apps/legal/*` (versioned ToS + consent model)
- `admin/src/views/auth/RegisterAgency.vue` (accept ToS)
- `web_app/src/views/auth/AcceptInvite.vue` (accept ToS)
- `content/legal/*.md` (source content)

## Test plan
**Manual:**
- Fresh signup → cannot proceed without ticking current ToS version
- Amend ToS → existing users prompted to re-accept on next login
- Request PAIA manual → PDF downloadable
