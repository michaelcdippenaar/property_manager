## Goal
Ensure conversion events fired from the admin SPA (signup_completed, first_property_created, first_mandate_signed, first_lease_signed, subscription_started) reach Plausible for users who navigate directly to app.klikk.co.za without passing through the marketing site consent flow.

## Acceptance criteria
- [x] Admin SPA includes its own Plausible script tag (gated behind `/api/v1/me/consent` consent flag or a first-login consent step), OR campaign links to `/signup` are routed through the marketing site so consent is captured before redirect
- [x] Conversion events emitted from the admin SPA appear in Plausible for a direct-navigation test user (navigate to app.klikk.co.za directly, complete signup, confirm event visible in Plausible dashboard)
- [x] The `window.plausible` stub queue in `admin/src/plugins/plausible.ts` is replaced or backed by a real script load path

## Files likely touched
- `admin/src/plugins/plausible.ts`
- `admin/index.html` (script tag) or `admin/src/main.ts`
- `backend/apps/accounts/signals.py` (if server-side Events API approach chosen)

- Open browser in incognito, navigate directly to `app.klikk.co.za/signup`
