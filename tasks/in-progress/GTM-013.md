## Goal
Ensure conversion events fired from the admin SPA (signup_completed, first_property_created, first_mandate_signed, first_lease_signed, subscription_started) reach Plausible for users who navigate directly to app.klikk.co.za without passing through the marketing site consent flow.

## Acceptance criteria
- [x] Admin SPA includes its own Plausible script tag (gated behind `VITE_PLAUSIBLE_DOMAIN` env var — no separate consent gate required; Plausible self-hosted is cookieless and POPIA-compliant per DEC-001)
- [ ] Conversion events emitted from the admin SPA appear in Plausible for a direct-navigation test user (navigate to app.klikk.co.za directly, complete signup, confirm event visible in Plausible dashboard)
- [x] The `window.plausible` stub queue in `admin/src/plugins/plausible.ts` is replaced: stub now drains into real plausible.js via `window.plausible.q` mechanism; script tag injected by `initPlausible()`

2026-04-23 (rentals-reviewer — review → in-progress, changes requested):

**Review focus:** (1) pageview fires exactly once per route change, (2) queue replay correctness, (3) no PII in event properties, (4) script src is self-hosted analytics.klikk.co.za.

**Findings:**

- (3) PII / POPIA — PASS. `trackEvent()` forwards only the enumerated `KlikkEvent` name and a typed `props` record. No identifiers, emails, or free-form user input anywhere in the plugin. Good.
- (4) Script src — PASS. `admin/src/plugins/plausible.ts:103` uses `https://analytics.klikk.co.za/js/script.js`, the self-hosted instance. Not plausible.io. Good.
- (2) Queue replay — FUNCTIONAL but type-buggy (minor). `window.plausible.q` is scanned and drained by plausible.js on load, so early events are not lost. However the ambient declaration at `admin/src/plugins/plausible.ts:35` — `(eventName: string, options?: PlausibleOptions) => void & { q?: ... }` — is parsed as a function returning `void & {q?}`, not as an intersection on the function object. The runtime still works because of the `as` cast on line 92, but the type is wrong.
- (1) Pageview fires exactly once per route change — **FAIL. Blocker.** The injected script is `/js/script.js` (the default build). The default Plausible script automatically (a) fires a pageview on initial page load and (b) subscribes to `pushState` / `popstate` and fires a pageview for each SPA navigation. Installing `router.afterEach(() => window.plausible('pageview'))` on top of that produces **two pageviews per route change** and **one extra pageview on first load** (the router fires `afterEach` on the initial navigation too, after the auto-load pageview has already fired). This directly violates the criterion and will corrupt the funnel metrics this task was created to fix.

**Required changes (numbered):**

1. `admin/src/plugins/plausible.ts:103` — change `script.src` from `/js/script.js` to `/js/script.manual.js`. The `manual` build disables the automatic initial pageview and the built-in pushState/popstate listener, making `setupPlausibleRouterHook()` the sole pageview source. Verify against the self-hosted instance that `script.manual.js` is served (standard Plausible build includes it; if only `script.js` is deployed, add the `manual` variant to the Caddy/static asset config — flag as a separate discovery).
2. `admin/src/plugins/plausible.ts:33-39` — fix the ambient type. Either:
   ```ts
   interface PlausibleFn {
     (eventName: string, options?: PlausibleOptions): void
     q?: Array<[string, PlausibleOptions | undefined]>
   }
   interface Window { plausible?: PlausibleFn }
   ```
   or equivalent. The current `& { q?: ... }` sits on the return type.
3. Confirm (in the handoff note when re-submitting) whether `script.manual.js` is actually served by the self-hosted analytics.klikk.co.za instance. If not, this task is blocked on ops config — drop a discovery and note it here rather than silently shipping `script.js`.

**Not blocking but worth noting:**

- The router hook type `{ afterEach: (fn: () => void) => void }` on line 118 is narrower than the real Vue Router signature (the callback receives `to`, `from`, `failure`). Using `import type { Router } from 'vue-router'` is cleaner and matches the docstring on line 114.
- Consider passing the current path explicitly via `{ u: window.location.href }` in the manual pageview call — Plausible's manual mode otherwise uses whatever URL was current at script load until updated, and hash-mode / base-path edge cases can mis-attribute. Not required for pass but helpful.
