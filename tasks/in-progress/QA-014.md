## Goal
Move all `--cov` flags into `pytest.ini` as the single source of truth so local and CI runs use identical coverage config and the maintenance trap of split configuration is eliminated.

## Acceptance criteria
- [ ] `pytest.ini` `addopts` includes `--cov=apps --cov-report=term-missing --cov-report=html:htmlcov --cov-report=xml:coverage.xml`
- [ ] CI `pytest` step in `.github/workflows/ci.yml` is plain `pytest --tb=short -q` with no `--cov` flags
- [ ] CI XML coverage report still uploaded to coverage service (confirm `coverage.xml` artifact path unchanged)
- [ ] Local `pytest` run produces HTML and XML reports

## Files likely touched
- `backend/pytest.ini`
- `.github/workflows/ci.yml`

## Test plan
**Manual:**
- Local: `cd backend && pytest --tb=short -q` — confirm `htmlcov/` and `coverage.xml` produced
- CI: green run with coverage report uploaded

**Automated:**
- CI pipeline passes
