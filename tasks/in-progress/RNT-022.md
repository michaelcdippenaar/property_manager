## Goal
Implement the Django backend for the AI Guide widget so the admin frontend can replace its in-process mock intent mapper with real Anthropic claude-3-5-haiku tool-use calls, with persistence and role-scoped action allowlisting.

## Acceptance criteria
- [ ] `backend/apps/ai/guide_views.py` — POST `/api/v1/ai/guide/` accepts `{message, portal}`, calls Anthropic claude-3-5-haiku with tool-use, returns `{reply, action}` structured payload
- [ ] `backend/apps/ai/guide_tools.py` — allowlisted actions (`create_property`, `list_leases`, `go_to_dashboard`, `view_maintenance`, etc.) scoped by portal role
- [ ] `backend/apps/ai/models.py` — `GuideInteraction` model: `user`, `portal`, `intent`, `action_taken`, `completed`, `created_at`
- [ ] `backend/config/settings/base.py` — `ANTHROPIC_API_KEY` and `ENABLE_AI_GUIDE` env vars documented
- [ ] `backend/config/urls.py` — `/api/v1/ai/guide/` wired
- [ ] Frontend mock swapped: `_mockIntentMapper` in `admin/src/stores/aiGuide.ts` replaced with `api.post('/ai/guide/', ...)` (comment stub already in place)
- [ ] Unit tests in `backend/apps/ai/tests/test_guide.py` cover: valid intent, unknown intent, role-scoped action blocked
