## Goal

Add a minimum viable move-in prep checklist to the lease detail view so agents have an in-app workflow for keys handover, utilities notification, tenant app invite, and welcome pack after a lease is signed.

## Acceptance criteria

- [ ] `content/product/features.yaml`: `tenant_onboarding.status` is corrected to reflect actual built state (set to `IN_PROGRESS` or `PLANNED` with `key_files: []` until this task ships)
- [ ] A "Move-in prep" section appears on the lease detail view once the lease status is `active`
- [ ] Checklist items (at minimum): Keys handed over, Utilities notified, Tenant app invite sent, Welcome pack delivered
- [ ] Checklist state is persisted per lease (new `move_in_checklist` JSON field on Lease model or dedicated MoveInChecklist model)
- [ ] Each checklist item records who completed it and when (agent name + timestamp)
- [ ] The checklist is visible to the agent and read-only to the owner in the owner portal
- [ ] `incoming_inspection` feature remains PLANNED; a placeholder card with "Coming soon" is shown below the checklist
- [ ] Migration included; no data loss on existing leases

## Files likely touched

- `content/product/features.yaml` (`tenant_onboarding` entry)
- `backend/apps/leases/models.py` (MoveInChecklist or json field)
- `backend/apps/leases/serializers.py`
