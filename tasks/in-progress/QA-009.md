## Goal
Exhaustively verify that every user role sees only what they should and can act only where they should. This is the single biggest source of multi-tenant leaks.

## Acceptance criteria
- [ ] Document the roles matrix: `staff`, `agency_admin`, `agent`, `managing_agent`, `estate_agent`, `owner`, `tenant`, `supplier` — scope (global, agency, property, lease) per role
- [ ] Test matrix: every role × every critical endpoint = expected status code
- [ ] Cross-agency leak tests: user from agency A cannot read/write agency B data (property, lease, mandate, payment, maintenance, document)
- [ ] Cross-property tenant leak: tenant A cannot access tenant B's lease/payments/chat
- [ ] Owner read-only scope: owner cannot mutate
- [ ] UI surfacing: admin SPA hides nav items + actions the role cannot perform (no ghost buttons)
- [ ] Findings addressed before ship; any residual gaps gate v1.0
