## Goal

When a tenant or owner completes signing a mandate or lease, the agent receives an in-app toast notification (regardless of which page they are on) and optionally an email alert, so signing events are never silently missed.

## Acceptance criteria

- [ ] When a `signer_completed` or `submission_completed` event fires (WebSocket or webhook), a global toast notification is shown in the admin SPA regardless of the agent's current route
- [ ] The toast includes the signer's name and document type (mandate / lease)
- [ ] Toast persists long enough to read (≥ 6 seconds) and is dismissible
- [ ] An email notification is sent to the agent's registered email address on signing completion (SES / Django email backend)
- [ ] Email includes: signer name, document type, property address, and a direct link to the relevant panel
- [ ] No duplicate notifications if the agent has the signing panel open (dedup by event ID)

## Files likely touched

- `admin/src/views/properties/MandateSigningPanel.vue`
- `admin/src/views/leases/ESigningPanel.vue`
- `admin/src/composables/useGlobalNotifications.ts` (new or extend existing)
- `admin/src/App.vue` (global toast listener)
- `backend/apps/leases/views.py` or webhook handler (trigger email on signing event)
- `backend/apps/leases/emails.py` (new email templates)

## Test plan

**Manual:**

- Check agent email inbox for signing notification
- Open signing panel before signing; confirm no duplicate toast
