## Goal
Audit and bump every "small" typography tier in the admin app up by one step in the scale so small text is no longer too small to read comfortably.

## Acceptance criteria
- [ ] Audit `admin/src/style/` (and any sibling token/config files) to identify all font-size variables and utilities used for "small" text — likely candidates: `--font-size-xs`, `--font-size-sm`, Tailwind `text-xs`, `text-sm` overrides, and component-specific small text in tables, badges, and captions.
- [ ] Bump every "small" tier up by 1 step in the existing scale (e.g. `text-xs` / 12px → 13px or 14px depending on scale; `text-sm` bumped similarly). Do not change base or large sizes.
- [ ] Verify visually that no text is clipped or overflows its container on: dashboard, properties list, lease detail, maintenance list, settings page.
- [ ] Run `npm test` inside `admin/` and confirm no snapshot regressions.

## Files likely touched
- `admin/src/style/tokens.css` (or `.scss` / `.ts` depending on convention)
- `admin/tailwind.config.js` (if Tailwind extends the size scale)
- Any per-component overrides that hardcode `font-size` in px for small/caption/badge text

## Test plan
**Manual:**
1. Open admin dev server (`npm run dev` in `admin/`).
2. Check dashboard — all badge, caption, and table cell text is legible at the new size.
3. Check properties list, lease detail, maintenance list, settings page — no overflow or clipping.
4. Spot-check dark/light mode if applicable.

**Automated:**
- `cd admin && npm test` — confirm no snapshot regressions introduced by the size change.
