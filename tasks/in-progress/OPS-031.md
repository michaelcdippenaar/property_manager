## Goal

Ensure a misconfigured production deploy fails loudly at startup rather than silently emailing customers `localhost` URLs for password resets, signing links, and supplier SMS.

## Acceptance criteria

- [ ] All five settings that currently fall back to localhost are validated at startup in production (non-`DEBUG`) mode:
  - `BASE_URL` (`maintenance/notifications.py:18`)
  - `SITE_URL` (`esigning/webhooks.py:497`)
  - `SIGNING_PUBLIC_APP_BASE_URL` (`accounts/views.py:419`)
  - Same setting in `accounts/admin_views.py:138, 154`
  - Same setting in `accounts/totp_views.py:368`
- [ ] Preferred approach: `AppConfig.ready()` or a `checks.py` Django system check that raises `ImproperlyConfigured` (or a critical check error) when `DEBUG=False` and the setting is falsy.
- [ ] Localhost fallback defaults are removed from the `getattr(...) or "http://localhost:..."` expressions.
- [ ] `DEBUG=True` (local dev) is unaffected — localhost defaults remain usable in development.
- [ ] No existing tests are broken.

## Files likely touched

- `backend/apps/maintenance/notifications.py` (line 18)
- `backend/apps/esigning/webhooks.py` (line 497)
- `backend/apps/accounts/views.py` (line 419)

## Test plan

**Manual:**

- Set all required vars; confirm server starts clean.
