## Goal
Establish a backend test baseline that every PR must pass: every Django app has pytest coverage, and the suite runs in under 5 minutes in CI.

## Acceptance criteria
- [ ] `backend/` full suite: all apps have a `tests/` directory with at least smoke coverage of models, views, serializers, and signals
- [ ] Target coverage: ≥65% line coverage on `apps/*` (baseline; not blocking on <65 in v1.0 but reported)
- [ ] Critical paths covered at ≥90%: leases/signing, payments, mandate, auth, RHA gate (RNT-SEC-007)
- [ ] `pytest backend` finishes < 5 min on CI (parallelise with `pytest-xdist` if needed)
- [ ] HTML coverage report uploaded as CI artifact
- [ ] Flaky tests inventoried and fixed or `@pytest.mark.flaky` quarantined with ticket

## Files likely touched
- `backend/apps/*/tests/*.py` (fill gaps)
- `backend/conftest.py`
- `backend/pytest.ini`
- `.github/workflows/ci.yml` (tie into OPS-001)

## Test plan
**Automated:**
- `cd backend && pytest --cov=apps --cov-report=html`
- CI run green
