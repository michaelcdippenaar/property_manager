## Goal
Replace the generic chat empty state with suggested starter prompts and a description of the AI assistant's scope so tenants understand and engage with the feature from day one.

## Acceptance criteria
- [ ] Empty state includes 2–3 tappable suggested prompts (e.g. "Report a repair", "When is my rent due?", "What does my lease say about pets?")
- [ ] Tapping a prompt opens a new conversation with that text pre-filled in the input
- [ ] Subtitle copy updated to describe scope: "Ask about your lease, maintenance requests, or anything about your property"
- [ ] FAB has a visible label or tooltip: "New conversation" (currently unlabelled icon only)
- [ ] Prompts reviewed and approved by UX/GTM before merge (choice of prompts is content, not just code)
- [ ] No regression on existing conversations list (prompts only shown when list is empty)

## Files likely touched
- `tenant/src/views/chat/ChatListView.vue` — line ~11, empty state section

## Test plan
**Manual:**
- Open chat tab as a tenant with no prior conversations; confirm prompts and updated subtitle visible
- Tap a suggested prompt; confirm new conversation opens with text pre-filled
- Create one conversation; confirm prompts disappear and normal conversation list renders
- Confirm FAB label visible on iOS and Android

**Automated:**
- Vitest snapshot or component test for `ChatListView` empty state branch
