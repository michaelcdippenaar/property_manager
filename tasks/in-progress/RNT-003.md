## Goal
Verify the native e-signing stack (post-DocuSeal) is rock-solid: public link signing, WebSocket real-time updates, audit trail, PDF layout, webhook fan-out.

## Acceptance criteria
- [ ] Public signing link works without login, on desktop + mobile
- [ ] Signature capture UI functions on touch devices
- [ ] Signed PDF matches draft layout (no shift) — verify via `esigning_pdf_layout_compare`
- [ ] `ESigningAuditEvent` populates with IP, user-agent, timestamp for every action
- [ ] Real-time status updates push via WebSocket to admin UI
- [ ] Resend flow works + re-signing invalidates prior signatures correctly
- [ ] `signed_pdf_file` stored and retrievable post-completion
