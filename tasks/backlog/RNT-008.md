## Goal
Ship the **agent** app as native iOS + Android builds to the App Store and Play Store by wrapping the Quasar PWA (served at `mobile-agent.klikk.co.za`) with Capacitor. One codebase, two native shells.

> **Launch strategy**: the Quasar **PWA** at `mobile-agent.klikk.co.za` must work as the primary mobile experience first. The native store apps are a fast-follow so App Store review timelines don't gate the v1.0 launch. `MIL-001` does NOT depend on RNT-008 — it depends on the PWA only.

## Acceptance criteria

### Capacitor wrap
- [ ] Capacitor added to Quasar agent project (`quasar mode add capacitor`)
- [ ] Web view loads the Quasar agent app with native-shell capabilities (camera, file picker, push tokens, biometrics for 2FA)
- [ ] Deep links + Universal Links / App Links configured for `app.klikk.co.za` and `mobile-agent.klikk.co.za`
- [ ] Native splash + icon (all required sizes for both platforms)
- [ ] Safe area handling for notched iOS + Android gesture nav

### iOS
- [ ] Xcode project builds signed release IPA
- [ ] `Info.plist`: minimal permission strings (camera — for property photos; photo library — for document upload; biometrics — for 2FA; push notifications)
- [ ] Privacy manifests (PrivacyInfo.xcprivacy) completed for Apple's April-2024 requirement
- [ ] TestFlight build uploaded; internal test group invited
- [ ] App Store listing: title, subtitle, description, keywords, screenshots (6.7" + 6.5" + iPad 12.9"), privacy labels, age rating

### Android
- [ ] Release `appbundle` builds + signs cleanly
- [ ] `AndroidManifest.xml`: permissions minimal and justified
- [ ] Data Safety form complete in Play Console
- [ ] Play Internal Testing track uploaded
- [ ] Play Store listing: title, description, screenshots (phone + 7" + 10" tablet), feature graphic, content rating

### Auth & sync
- [ ] Same JWT flow as web (store in secure enclave / Keystore, not SharedPreferences / UserDefaults)
- [ ] 2FA TOTP works via biometric unlock (ties to `RNT-SEC-003`)
- [ ] Push via FCM + APNs (ties to `RNT-QUAL-007`)

### Crash + monitoring
- [ ] Sentry Capacitor integrated (ties to `OPS-002`)
- [ ] Release + dist tagged per build

**Manual:**

- Install Play Internal build on a real Android device → same smoke test
- Kill + reopen app → session persists (JWT refresh works from secure storage)
- Receive push on both devices → deep-link opens correct screen

**Automated:**
