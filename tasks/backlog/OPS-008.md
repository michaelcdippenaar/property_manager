## Goal
Know within 2 minutes when production is down. Before first client, we need external uptime checks, deep healthchecks, and an on-call path.

## Acceptance criteria
- [ ] `/health/` endpoint: basic liveness (DB connect, Redis connect, Gotenberg reachable, disk space)
- [ ] `/health/deep/` endpoint (auth-gated): runs a full lease-render probe every 15 min
- [ ] External uptime monitor (BetterStack / UptimeRobot / HealthChecks.io) hitting every surface + deep health every 5 min
- [ ] Alerts: SMS to MC + email to ops list when down > 2 min
- [ ] Status page: public `status.klikk.co.za` with incident history
- [ ] On-call rotation doc in `docs/ops/on-call.md` (v1.0 = MC solo, named)

## Files likely touched
- `backend/apps/health/views.py`
- `backend/config/urls.py`
- `docs/ops/on-call.md` (new)

## Test plan
**Manual:**
- Stop DB container → /health returns 503 → alert fires within 2 min
- Stop Gotenberg → deep probe fails → alert
