## Goal
Stand up a CI/CD pipeline so every PR is linted/tested automatically and every main-branch merge deploys to staging. Without this, regressions land silently and deploys are manual + error-prone.

## Acceptance criteria
- [ ] `.github/workflows/ci.yml`: runs on PR — black/ruff, eslint, `pytest backend`, `npm test` for admin + web_app, `flutter analyze`
- [ ] `.github/workflows/staging-deploy.yml`: on merge to main → build Docker images, push to registry, SSH + `docker compose pull && up -d` on staging host
- [ ] Branch protection: PR requires green CI to merge
- [ ] Status badge in repo README
- [ ] Secrets managed as GitHub Actions secrets (SSH key, registry creds, env)
- [ ] Rollback: previous image tag kept, documented one-command rollback

## Files likely touched
- `.github/workflows/ci.yml` (new)
- `.github/workflows/staging-deploy.yml` (new)
- `backend/Dockerfile`, `admin/Dockerfile`, `web_app/Dockerfile`
- `docs/ops/ci-cd.md` (new)

## Test plan
**Manual:**
- Open PR → CI runs and reports status → merge → staging deploys → new commit SHA visible at staging URL
- Trigger rollback → previous SHA live
