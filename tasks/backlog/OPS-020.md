## Goal
Remove the inline staging server SSH password from the committed runbook and disable password-based SSH authentication on the staging server to close the credential leak.

## Acceptance criteria
- [ ] Inline password comments removed from `deploy/DEVOPS.md` (lines 19 and 276)
- [ ] Staging server SSH password rotated (MC performs this server-side action)
- [ ] `PasswordAuthentication no` set in `/etc/ssh/sshd_config` on the staging server — key-based auth only
- [ ] MC's personal SSH public key added to `~/.ssh/authorized_keys` on the staging server (alongside the deploy key from OPS-012)
- [ ] `grep -n "password:" deploy/DEVOPS.md` returns zero matches

## Files likely touched
- `deploy/DEVOPS.md`
- Server-side: `/etc/ssh/sshd_config` (human action required)

## Test plan
**Manual:**
- `ssh mc@102.135.240.222` with old password — expect rejection
- `ssh -i ~/.ssh/id_ed25519 mc@102.135.240.222` — expect success
