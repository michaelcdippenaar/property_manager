## Goal
Eliminate the split pytest coverage configuration between `pytest.ini` and the CI workflow by moving all flags (including `--cov-report=xml`) into `pytest.ini` as the single source of truth.

## Acceptance criteria
- [ ] `backend/pytest.ini` `addopts` includes all coverage flags: `--cov=apps --cov-report=term-missing --cov-report=html:htmlcov --cov-report=xml:coverage.xml`
- [ ] `.github/workflows/ci.yml` pytest step is plain `pytest --tb=short -q` with no `--cov` flags
- [ ] `cd backend && pytest --tb=short -q` locally produces `htmlcov/` and `coverage.xml`
- [ ] CI coverage upload step (if any) continues to find `coverage.xml`

## Files likely touched
- `backend/pytest.ini`
- `.github/workflows/ci.yml`

## Test plan
**Manual:**
- Run `cd backend && pytest --tb=short -q` locally — confirm `coverage.xml` is generated
- Push a PR and verify CI coverage report is still uploaded/visible
