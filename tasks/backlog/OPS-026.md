## Goal

Provision the AWS infrastructure (S3 bucket + IAM user) documented in
`docs/ops/backup-restore.md`, run one complete backup cycle on staging, execute
the restore drill procedure, and record the result in `docs/ops/restore-drills.md`.

This is the live-infra complement to OPS-003, which delivered all code and
documentation. The drill cannot be run until the S3 bucket and IAM user exist.

## Acceptance criteria

- [ ] S3 bucket `klikk-backups-af` created in `af-south-1` with versioning enabled
- [ ] S3 lifecycle rules configured (30d daily, 365d monthly, media non-current → Glacier after 90d)
- [ ] IAM user `klikk-backup` created with write-only policy from `docs/ops/backup-restore.md § 2`
- [ ] IAM credentials stored in Bitwarden under "Klikk → Backup IAM credentials"
- [ ] age keypair generated; private key in Bitwarden "Klikk → Backup age private key"; public key in `backend/.env.secrets` as `BACKUP_ENCRYPTION_KEY`
- [ ] `backup.sh` runs to completion on staging (exit 0, encrypted dump visible in S3)
- [ ] Restore drill executed per `docs/ops/backup-restore.md § 8`; row counts match
- [ ] Drill result recorded in `docs/ops/restore-drills.md` with a real timestamped entry (replacing the PENDING placeholder)
- [ ] Monthly cron reminder (`drill_reminder` container) confirmed running
