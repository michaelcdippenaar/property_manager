## Goal
Ship the **tenant** app as native iOS + Android builds by wrapping the Quasar tenant PWA (at `mobile-tenant.klikk.co.za`) with Capacitor. Replaces the previously deleted RNT-006 (Flutter store submission).

> **Launch strategy**: same as RNT-008 — the Quasar tenant PWA ships at v1.0 launch; native apps are a fast-follow. `MIL-001` does not depend on RNT-009.

## Acceptance criteria

### Capacitor wrap
- [ ] Capacitor added to Quasar tenant project
- [ ] Web view loads tenant PWA with native capabilities: camera (maintenance photos), file picker (document upload), biometrics (optional 2FA per DEC-011), push tokens
- [ ] Deep links for `tenant.klikk.co.za` and `mobile-tenant.klikk.co.za`
- [ ] Native splash + icon (all required sizes)

### iOS
- [ ] Xcode project builds signed release IPA
- [ ] `Info.plist`: permission strings for camera, photo library, push, biometrics — human-readable, accurate
- [ ] `PrivacyInfo.xcprivacy` complete
- [ ] TestFlight uploaded; internal testers invited (including MC + first-client contacts from DEC-007)
- [ ] App Store listing: title, subtitle, description, keywords, screenshots, privacy labels

### Android
- [ ] Release `appbundle` builds + signs
- [ ] Permissions minimal and justified
- [ ] Play Data Safety form complete
- [ ] Play Internal Testing track uploaded
- [ ] Play Store listing complete

### Auth & sync
- [ ] Magic-link / password flow works (per final DEC on tenant auth)
- [ ] Optional 2FA via biometrics (per DEC-011: conditional + always-on opt-in)
- [ ] POPIA s23 / s24 self-service accessible from native app (per RNT-SEC-006)

### Crash + monitoring
- [ ] Sentry Capacitor integrated (ties to `OPS-002`)
