## Goal
Before we take on a paying client we need a tested, automated backup/restore path for everything a client entrusts us with: Postgres data + encrypted media files.

## Acceptance criteria
- [ ] Postgres: nightly `pg_dump --format=custom` encrypted with age/gpg → S3 (af-south-1, lifecycle: 30 days standard, then Glacier)
- [ ] Media (including Vault `.enc` blobs): nightly `rclone sync` or `aws s3 sync` to S3, versioning on
- [ ] Retention: 30 daily + 12 monthly
- [ ] Restore drill documented: bring up a fresh Postgres + restore latest dump + checksum compare
- [ ] Drill executed monthly (cron reminder) with timestamped log in `docs/ops/restore-drills.md`
- [ ] Backup failure alerts to Slack/email
- [ ] IAM: dedicated backup user, write-only (+ list), not delete
