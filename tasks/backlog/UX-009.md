## Goal

Remove the live "Advertise unit" action from the property detail Actions dropdown until the `property_advertising` feature ships, so agents never hit a dead end at the most critical step in the first rental cycle.

## Acceptance criteria

- [ ] "Advertise unit" is hidden (preferred) or disabled with a "Coming soon" tooltip in `PropertyDetailView.vue` while `property_advertising` has `status: PLANNED` in `features.yaml`
- [ ] No other PLANNED feature appears as an active nav item or action button anywhere in the admin SPA
- [ ] A feature-gating convention is documented (comment in code or brief ADR) so future PLANNED features are gated by default
- [ ] Change is covered by a Cypress or Vitest component test asserting the button is absent / disabled when the feature flag is PLANNED

## Files likely touched

- `admin/src/views/properties/PropertyDetailView.vue` (~line 98)
- `admin/src/composables/useFeatureFlags.ts` (new, or extend existing)
- `content/product/features.yaml` (reference only — no status change needed here)
