## Goal
Replace the invalid `<label for="invite-email"><p id="invite-email">` association in `AcceptInviteView.vue` with a spec-compliant labelable element so screen readers correctly announce the email field label (WCAG 1.3.1).

## Acceptance criteria
- [ ] `<p id="invite-email">` replaced with `<input type="email" id="invite-email" :value="invite.email" readonly tabindex="-1">` (or equivalent labelable element)
- [ ] Alternatively, `<label>` is removed and a visible label with `aria-describedby` or visually-hidden `<span>` is used
- [ ] VoiceOver announces "Email" when focus reaches the email display

## Files likely touched
- `admin/src/views/auth/AcceptInviteView.vue` (lines 48-49)
