## Goal
Wire Sentry across every surface so production errors reach us with full context (user role, agency, request, breadcrumbs) instead of dying silently.

## Acceptance criteria
- [ ] Backend: `sentry-sdk[django]`, DSN from env, `send_default_pii=False`, release = git SHA
- [ ] Admin SPA: `@sentry/vue`, DSN from env, source maps uploaded in CI
- [ ] Tenant web: same pattern
- [ ] Mobile Flutter: `sentry_flutter`, release + dist tagged
- [ ] Mobile Quasar: `@sentry/capacitor`
- [ ] PII scrubbing: strip id_number, passport, bank account from payloads (hook)
- [ ] Alerts: Slack/email on new issue, regression, error-rate spike
- [ ] Per-tier quota respected (dev/staging/prod projects)

## Files likely touched
- `backend/config/sentry.py`
- `backend/config/settings.py`
- `admin/src/main.ts`, `admin/vite.config.ts` (source maps)
- `web_app/src/main.ts`
- `mobile/lib/main.dart`
- `agent-app/src/main.ts`

## Test plan
**Manual:**
- Trigger a forced exception in each surface → event lands in Sentry project with correct release + environment + user context
- PII scrubbing: id_number in request body → Sentry payload has redaction
