## Goal
Make every marketing action measurable from impression → click → signup → activation → paid. Without this the campaigns in GTM-003/004/005 are pure guesswork.

## Acceptance criteria
- [x] Analytics stack chosen and installed:
  - Marketing + product: Plausible (privacy-first, POPIA-friendly, af-south-1 if available) OR PostHog (with cookie-less config)
  - Conversion events: signup_started, signup_completed, first_property_created, first_mandate_signed, first_lease_signed, subscription_started
- [x] UTM convention documented in `content/marketing/utm-convention.md` (utm_source, medium, campaign, content, term — controlled vocab)
- [x] Every outbound link in website, emails, sales decks uses correct UTMs (linter / style guide)
- [x] Attribution: first-touch + last-touch + linear — pick one primary model, compute in analytics
- [x] Per-channel dashboard: impressions, clicks, signups, activations, CAC, payback (`docs/marketing/attribution-dashboard.md`)
- [x] Cohort view: week-over-week signup-to-activation funnel
- [x] Weekly review ritual documented (who looks at what, when)
- [x] POPIA: consent banner + cookie policy aligned; no PI leaked to third-party analytics
- [x] Ties into OPS-002 Sentry releases (tag releases with campaign context where relevant)

## Files likely touched
- `website/src/layouts/BaseLayout.astro` (analytics tag)
- `admin/src/main.ts` (in-app event emission)
- `web_app/src/main.ts`
- `content/marketing/utm-convention.md` (new)
- `docs/marketing/attribution-dashboard.md` (new)
- `backend/apps/analytics/*` (optional first-party event capture)

## Test plan
**Manual:**
- Click a campaign link with UTMs → analytics shows the session with correct params
- Complete a signup flow → all funnel events present in order
- Check POPIA: consent declined → no analytics events fired
