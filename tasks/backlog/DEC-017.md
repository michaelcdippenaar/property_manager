## Question
`TOTP_REQUIRED_ROLES` does not include `Role.ACCOUNTANT` or `Role.VIEWER`. Both roles exist and can access financial/lease data in the admin SPA. Should they be required to complete 2FA enrollment before accessing the platform?

## Options

**Option A — Add ACCOUNTANT and VIEWER to TOTP_REQUIRED_ROLES (recommended)**
- Pros: consistent with intent of RNT-SEC-003 (2FA for any role touching financial data); accountants see rent rolls and payment records; reduces risk if accountant accounts are compromised.
- Cons: friction for read-only VIEWER accounts; requires a migration and updated test suite.

**Option B — Add ACCOUNTANT only; leave VIEWER exempt**
- Pros: balances friction vs. risk — viewers are read-only, accountants modify financials.
- Cons: VIEWER can still see rent rolls; a compromised viewer account leaks financial data without 2FA.

**Option C — Keep current state (neither required)**
- Pros: no friction, no migration.
- Cons: inconsistent with stated RNT-SEC-003 acceptance criteria; financial data accessible without 2FA.

## Recommendation
Option A. Accountants have write access to financial records; viewers have read access to the same data. Both warrant 2FA in a multi-tenant property management platform subject to POPIA.
