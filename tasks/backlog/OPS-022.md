## Goal
Eliminate the ~10-20s MJML re-download on every CI push by adding an npm cache to the `email-templates` job's `actions/setup-node` step.

## Acceptance criteria
- [ ] `actions/setup-node@v4` step in the `email-templates` job includes `cache: 'npm'` (or mjml pinned as a devDependency so standard `npm ci` cache applies)
- [ ] CI `email-templates` job cache hit confirmed in GitHub Actions run logs on the second push
