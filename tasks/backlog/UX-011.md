## Goal
Return the tenant to the app with a confirmation state after they sign their lease in the external signing tab, so they know the action completed successfully.

## Acceptance criteria
- [ ] After signing, the external signing flow redirects back to the tenant portal with a detectable query param (e.g. `?signed=1`) or the backend webhook triggers an in-app state update
- [ ] On return, the pending-signing CTA on the Home dashboard is dismissed without requiring a manual page reload
- [ ] Tenant sees an in-app confirmation: "Lease signed — awaiting countersignature from your agent"
- [ ] If redirect-back is not feasible for web, a polling loop (max 5s interval) fires after the signing tab is opened and clears the CTA on confirmation
- [ ] On native (Capacitor Browser), the `browserFinished` event is used to trigger the same state refresh
- [ ] No regression: agents still receive the signing completion webhook as before

## Files likely touched

- `tenant/src/views/home/HomeView.vue` — line ~152, `signingCta` flag refresh logic
