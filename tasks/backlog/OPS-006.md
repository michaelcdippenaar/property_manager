## Goal
Finalise the full production subdomain map so every Klikk surface is reachable at a stable URL with valid TLS, correct auth records, and sensible redirects. All subdomains are resolved per `DEC-013` (answered 2026-04-22).

## Canonical subdomain map

| Subdomain | Role | Type |
|---|---|---|
| `klikk.co.za` | Marketing site (Astro) | Canonical |
| `app.klikk.co.za` | Admin SPA (Vue 3, `admin/`) | Canonical agent URL |
| `agent.klikk.co.za` | → `app.klikk.co.za` | 301 redirect |
| `tenant.klikk.co.za` | Tenant web app (`web_app/`) | Canonical tenant URL |
| `mobile-agent.klikk.co.za` | Quasar agent PWA | Canonical |
| `mobile-tenant.klikk.co.za` | Quasar tenant PWA | Canonical |
| `api.klikk.co.za` | Django API | Canonical |
| `backend.klikk.co.za` | → `api.klikk.co.za` | 301 redirect (sunset after 90 days) |
| `status.klikk.co.za` | Uptime status page | Canonical (per `OPS-008`) |
| `mail.klikk.co.za` | AWS SES transactional sender | Isolated SPF/DKIM/DMARC |
| `www.klikk.co.za` | → `klikk.co.za` | 301 redirect |

## Acceptance criteria

### TLS + Caddy
- [ ] Every canonical subdomain has a valid Let's Encrypt cert via Caddy auto-HTTPS (renewal automatic, monitored)
- [ ] Redirects (`agent.`, `backend.`, `www.`) use `redir {scheme}://<canonical>{uri} permanent`
- [ ] TLS-A+ rating (Qualys SSL Labs) on every canonical surface
- [ ] HSTS on all canonical surfaces; preload submission after 30 days of stable cert

### DNS records
- [ ] A / AAAA records correct for every subdomain pointing at the production host
- [ ] CNAME for `www` → apex
- [ ] Document every record in `docs/ops/dns.md`

### Email auth (critical: dual-domain setup)
- [ ] `klikk.co.za` apex keeps Google Workspace SPF/DKIM/DMARC — do NOT touch; `mc@klikk.co.za` and other human mailboxes unaffected
- [ ] `mail.klikk.co.za` gets AWS SES SPF + DKIM (CNAME records from SES console)
- [ ] `_dmarc.mail.klikk.co.za` → `v=DMARC1; p=quarantine; rua=mailto:dmarc@klikk.co.za; pct=100` at launch; move to `p=reject` after 30 days if reports clean
- [ ] SES reverse-DNS (PTR) set where available
- [ ] All transactional email from `noreply@mail.klikk.co.za` (or `notifications@mail.klikk.co.za`); reply-to can be `hello@klikk.co.za`
- [ ] mail-tester.com score ≥ 9/10 from SES send

### Django config
- [ ] `ALLOWED_HOSTS` includes: `api.klikk.co.za`, `backend.klikk.co.za` (legacy), `app.klikk.co.za`, `tenant.klikk.co.za`, `mobile-agent.klikk.co.za`, `mobile-tenant.klikk.co.za`
- [ ] `CSRF_TRUSTED_ORIGINS` includes all agent + tenant surfaces (app, agent redirect, mobile-agent, tenant, mobile-tenant)
- [ ] `CORS_ALLOWED_ORIGINS` includes those same surfaces — no wildcards

### CDN
- [ ] Static assets behind Cloudflare (or S3 + CloudFront) with long-cache + hashed filenames
- [ ] Cache-Control headers correct per asset type
- [ ] Origin protected (Cloudflare Authenticated Origin Pulls or equivalent)

### Monitoring
- [ ] UptimeRobot (per `OPS-008`) hitting: klikk.co.za, app., tenant., api., mobile-agent., mobile-tenant., status. — every 5 min
- [ ] Alerts fire on: TLS expiry < 14 days, 5xx rate > 1%, down > 2 min
