## Question
Should `Role.ACCOUNTANT` and `Role.VIEWER` be added to `TOTP_REQUIRED_ROLES`, requiring those users to complete 2FA setup before accessing the admin SPA?

## Context
RNT-SEC-003 added mandatory 2FA for agent and admin roles. `ACCOUNTANT` and `VIEWER` were not included, but:
- **ACCOUNTANT** has read access to financial data (rent rolls, payment records, ledger)
- **VIEWER** has broad read access to property and lease data including tenant PII

Both roles are currently reachable with username + password only.

## Options

### Option A — Add both to TOTP_REQUIRED_ROLES (recommended)
**Pros:** Consistent security posture; ACCOUNTANT financial data is higher-risk than some agent actions; POPIA principle of least-privilege access hardening.
**Cons:** Friction for accountants/viewers who use the app infrequently; requires a migration and a communication to existing users of those roles.

### Option B — Add ACCOUNTANT only, not VIEWER
**Pros:** Balances risk vs friction; VIEWER has no write access so breach impact is lower.
**Cons:** VIEWER still sees tenant PII and lease data; inconsistent security posture.

### Option C — Neither (status quo)
**Pros:** No user friction; no migration required.
**Cons:** Financial and PII data accessible with password only; inconsistent with stated intent of RNT-SEC-003.

## Recommendation
**Option A.** ACCOUNTANT access to financial records warrants the same 2FA gate as agents. VIEWER accesses enough PII that the incremental friction of 2FA is justified.
