## Goal
Harden rent payment reconciliation for launch: correct matching of payments to tenants via unique reference, arrears flagging accuracy, receipt generation, partial payments.

## Acceptance criteria
- [ ] Payment with correct unique reference auto-matches to tenant + lease
- [ ] Payment with wrong reference flagged for manual reconciliation
- [ ] Partial payments tracked, don't falsely mark rent paid
- [ ] Arrears correctly flagged when rent overdue by N days (configurable)
- [ ] Receipts generate on full payment
- [ ] Owner dashboard reflects payment state in real time
- [ ] No off-by-one errors around month boundaries or leap days

## Files likely touched
- `backend/apps/leases/models.py` (payment models)
- `backend/apps/leases/` (payment-related views)
- Owner dashboard views

## Test plan
**Automated:**
- `cd backend && pytest apps/leases/tests/ -k payment -v`

**Manual:**
- Simulate: (1) exact match, (2) wrong reference, (3) partial, (4) late payment crossing arrears threshold
