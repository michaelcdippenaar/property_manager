## Question
`content/product/pricing.yaml` defines four tiers: `free`, `pro`, `enterprise`, `custom`. But the file's own header comment, all `available_on` arrays, positioning.md, competitive-matrix.md, icp.md, objections.md, and demo-flow.md all reference an "Agency" tier with invented per-unit pricing (R39–R49/unit). No `agency:` key exists in pricing.yaml. Should we:

**Option A — "Agency" is the correct commercial name; rename `enterprise` → `agency` in pricing.yaml**
- Pros: aligns the YAML with all the marketing copy; "Agency" is a better market-facing name than "Enterprise" for a property management SaaS
- Cons: minor code change to any place that checks the `enterprise` key

**Option B — Enterprise is correct; remove all "Agency" references from copy and pricing.yaml comments**
- Pros: no tier rename, simpler
- Cons: all sales/marketing files need auditing and updating

**Option C — Both exist: keep `enterprise` and add a new `agency` tier at a different price point**
- Pros: potential upsell opportunity for multi-branch agencies
- Cons: adds complexity, requires a new pricing decision

**PM recommendation:** Option A — "Agency" is a better commercial label for the R4,000/month top tier targeting multi-property agents. Rename `enterprise` → `agency` in pricing.yaml and propagate. Separately, the per-unit pricing fiction (R49/unit) in the sales files must be removed regardless — it has no basis in the actual pricing model.
