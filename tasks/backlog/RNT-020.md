## Goal

Embed a natural-language guide agent in the agent and owner portals that can navigate users to the right page, highlight the correct CTA, and optionally perform the action on their behalf — reducing time-to-first-value for new users and cutting support load.

## Acceptance criteria

- [ ] Chat widget (`AIGuide.vue`) mounted in `AppLayout.vue` for both agent and owner portals; widget is collapsible and non-blocking.
- [ ] User can type a natural-language intent (e.g. "I want to create a property") and the agent responds with navigation and/or highlights the relevant UI element.
- [ ] Two modes supported: **Walk-through** (highlight next click, user acts) and **Do-it-for-me** (agent pre-fills / submits forms via allowlisted RPA actions).
- [ ] Backend intent mapper (`backend/apps/ai/guide_views.py`) receives user message, determines intent, and returns a structured action payload (route, element selector, pre-fill data, confirmation required flag).
- [ ] Tool-use loop: backend agent has an allowlisted set of actions (create property, list leases, go to dashboard, view maintenance, etc.) — no action outside the allowlist can be triggered.
- [ ] Actions are permission-scoped: agent-portal users see agent-role tools only; owner-portal users see owner-role tools only.
- [ ] Telemetry: each guide interaction (intent, action taken, completion/abandonment) is logged to a `GuideInteraction` model for UX analytics.
- [ ] Anthropic API key injected via env var (`ANTHROPIC_API_KEY`); never hardcoded.
- [ ] `useAIGuide.ts` composable exposes `{ sendMessage, highlight, clearHighlight, isOpen, toggle }` to the rest of the frontend.
- [ ] `aiGuide` Pinia store holds conversation history, highlight state, and router-navigation state.
- [ ] Unit tests for intent mapper (mock Anthropic responses); E2E smoke test for walk-through mode.
- [ ] Feature flag (`ENABLE_AI_GUIDE` env var, default `false`) so the widget can be toggled without a deploy.

## Test plan

**Manual:**
1. Open agent portal → click guide widget → type "create a new property" → confirm agent navigates to the correct page and highlights the "Add Property" button.
2. Switch to walk-through mode → confirm step-by-step highlights appear in sequence.
3. Switch to do-it-for-me mode → confirm agent pre-fills the property creation form.
4. Log in as owner → confirm only owner-scoped tools are available in the guide.
5. Disable `ENABLE_AI_GUIDE` → confirm widget is not rendered.

**Automated:**
- `cd backend && pytest apps/ai/tests/test_guide.py -v`
