## Goal

Deliver a channel-abstracted OTP service backed by email (AWS SES, already configured) that unblocks registration, password reset, and sensitive-change flows at launch, with SMS swappable in as a zero-rework drop-in adapter once WinSMS/Panacea onboarding clears.

## Acceptance criteria

- [ ] `apps/accounts/otp/service.py` exposes `OTPService.send(user, purpose: str)` and `OTPService.verify(user, code: str, purpose: str) -> bool`
- [ ] `apps/accounts/otp/channels/base.py` defines a `Channel` ABC with `send(recipient, code, context) -> None`
- [ ] `apps/accounts/otp/channels/email.py` implements the email channel using the existing Django email backend (SES in prod, console in dev); template at `apps/accounts/templates/otp/email.txt` â€” plain text, code prominent, "expires in 5 min", POPIA footer
- [ ] `apps/accounts/otp/channels/sms.py` is a stub that raises `NotImplementedError("SMS channel pending WinSMS/Panacea integration")` â€” interface-compatible so the provider can be wired later without touching callers
- [ ] Channel selection driven by `settings.OTP_CHANNELS` (ordered list; first-working wins); default `["email"]` for v1
- [ ] 6-digit numeric code, 5-minute TTL, hashed at rest â€” never store plaintext; use `make_password`/`check_password` or HMAC-SHA256 with per-user salt
- [ ] Rate limits: max 5 OTP issues per user per hour; max 3 verify attempts per code (lock after 3 fails, require new code)
- [ ] Per-purpose code scoping: a code issued for `purpose="registration"` cannot verify a `purpose="password_reset"` request â€” prevents cross-flow replay
- [ ] `OTPCode` model: `user`, `purpose`, `code_hash`, `created_at`, `expires_at`, `consumed_at`, `attempt_count`, `channel_used` â€” migration included
- [ ] `OTPAuditLog` rows written on every issue / verify-success / verify-fail (POPIA audit trail) â€” piggyback on existing audit app if one exists, otherwise new model
- [ ] Unit tests cover: service send/verify, email channel, rate limiting, replay prevention, TTL expiry, cross-purpose isolation â€” target 95%+ coverage on the `otp` package
