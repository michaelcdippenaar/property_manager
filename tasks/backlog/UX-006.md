## Goal

Instrument the admin SPA with Microsoft Clarity (POPIA-gated) and produce a complete UAT session kit so MC's wife can complete a structured first-tester run before Irma Knipe Properties outreach.

## Acceptance criteria

- [ ] Microsoft Clarity snippet integrated into `admin/` via `admin/src/composables/useClarity.ts`, controlled by `VITE_ENABLE_CLARITY` env var — OFF in dev/CI, ON only on staging
- [ ] Clarity snippet loads ONLY after user accepts a POPIA-compliant consent banner: "We record this session to improve Klikk. Data: clicks, mouse movement, screen content with sensitive fields masked. You can opt out anytime."
- [ ] Consent banner implemented in `admin/src/components/ConsentBanner.vue`, mounted in `admin/src/App.vue`
- [ ] Sensitive inputs masked via `data-clarity-mask` attribute: password fields, ID numbers, bank detail fields, lease PDF viewer iframe
- [ ] `admin/.env.staging.example` documents `VITE_ENABLE_CLARITY=true`
- [ ] `marketing/ux/uat-protocol-v1.md` created with: 6–10 task scenario script covering core rental flow (property add → unit → tenant → lease → e-sign → payment → maintenance), observer note template, and standard 10-question SUS questionnaire
- [ ] `marketing/ux/uat-consent-form.md` created — one-page POPIA consent form for tester to sign before session
- [ ] `marketing/ux/uat-runbook.md` created — 30-min session run-book: how to start, facilitator script, what to say, how to review replay afterward
- [ ] NOT in scope: PostHog setup (separate task post-prod-launch)

## Test plan

**Manual:**
- Local `npm run dev` (no `VITE_ENABLE_CLARITY` set): no consent banner appears, no Clarity script tag in DOM
- Staging build with `VITE_ENABLE_CLARITY=true`: consent banner appears on first load; Clarity script only injected into DOM after user clicks Accept
- Dismiss banner (Decline): Clarity is NOT loaded; preference persisted in localStorage so banner does not re-appear on reload
- Open a form with a password field or ID number input: confirm the field carries `data-clarity-mask` and does not surface in Clarity recording (verify via Clarity session replay in staging project)
- Lease PDF viewer: confirm iframe/container carries `data-clarity-mask`
- Wife dry-run: complete all scenario tasks in `uat-protocol-v1.md`, facilitator scores SUS questionnaire, confirm Clarity replay is readable

**Automated:**
- No automated tests required for this task; Clarity integration is runtime/env-gated
