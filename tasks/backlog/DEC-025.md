## Question for MC

Should the in-app AI guide be permitted to call write (create / update / delete) API endpoints directly on behalf of the logged-in user, or should it remain read + navigate only?

## Context

RNT-AI-002 requires an action-execution layer so the AI guide can fulfil requests like "Create new property for me with the name 18 Doctor Malan Road" end-to-end. Before implementation begins, three sub-questions must be answered:

1. **Allowed?** Is the AI allowed to POST/PATCH/DELETE via role-bounded tools, or is navigation + highlighting the permanent ceiling?
2. **Audit trail?** If yes — do we log AI-initiated mutations separately (e.g. `created_by_ai: true` flag, separate audit table)?
3. **Confirmation UX?** Does the AI need explicit user confirmation ("About to create property '18 Doctor Malan Road' — confirm?") before executing each write, or can it act silently in auto mode?

## Options

### Option A — Read + navigate only (status quo)
The AI guide never calls write endpoints. It navigates, highlights, and coaches. Users always perform writes themselves.
- **Pro:** Zero risk of unintended mutations; no audit complexity.
- **Con:** "Auto" mode is misleading — does not fulfil user expectation set by the feature name.

### Option B — Action execution with mandatory confirmation
AI can call write endpoints but must show a confirmation card ("I'm about to create property X — OK?") before each mutation.
- **Pro:** Meets user expectation; safe UX; easy to audit (user click is the audit event).
- **Con:** Extra round-trip; confirmation fatigue for power users.

### Option C — Action execution, silent in auto mode
AI acts immediately in auto mode; shows a summary card after completion.
- **Pro:** Fastest UX; matches "autopilot" mental model.
- **Con:** Higher risk of accidental mutations; requires robust undo or soft-delete.

## Recommendation

**Option B.** Confirmation-gated writes give MC the "it does it for me" experience while keeping mutations deliberate. Audit trail = the confirmation click timestamp + `ai_guide` source tag on the created resource. Implement undo (soft-delete + restore) as a follow-on.
