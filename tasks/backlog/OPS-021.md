## Goal
Prevent silent breakage of conditional email rendering when `base.mjml` is recompiled by creating an automated build script that applies the Django `{% if %}` patches after MJML compilation.

## Acceptance criteria
- [ ] `scripts/compile_email_templates.sh` (or `Makefile` target) runs `npx mjml base.mjml -o compiled/base.html` and then applies `{% if cta_url %}` / `{% if note %}` wrappers via a `sed` or Python patch step
- [ ] CI check (pre-commit hook or GitHub Actions step) fails if `compiled/base.html` is out of date relative to `base.mjml` without the conditional blocks present
- [ ] Compile step documented in `backend/apps/notifications/NOTIFICATIONS.md`
- [ ] `npm run build-emails` (or equivalent) produces a correctly patched `compiled/base.html` from a clean `base.mjml`

## Files likely touched
- `scripts/compile_email_templates.sh` (new)
- `Makefile` or `.github/workflows/ci.yml`
- `backend/apps/notifications/NOTIFICATIONS.md`

## Test plan
**Manual:**
- Make a cosmetic change to `base.mjml`, run the compile script, verify `compiled/base.html` contains the `{% if cta_url %}` block
