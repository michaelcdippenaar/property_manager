## Goal
Get the Flutter tenant app into a shippable state for App Store (iOS) and Play Store (Android): metadata, icons, splash, permissions manifest, release build, store listings.

## Acceptance criteria
- [ ] App icon and splash screen final (iOS + Android assets at all required sizes)
- [ ] iOS: Info.plist permissions strings human-readable and accurate
- [ ] Android: manifest permissions minimal and justified
- [ ] Release build signs cleanly for both platforms
- [ ] Privacy policy + terms URLs resolve + POPIA-compliant copy
- [ ] App Store / Play Store metadata drafted (title, description, keywords, screenshots)
- [ ] Deep links / universal links tested
- [ ] Crash reporting (Sentry or equivalent) configured

## Files likely touched
- `mobile/ios/`, `mobile/android/`
- Store metadata (likely new files under `mobile/store_metadata/`)
- Icon + splash assets

## Test plan
**Manual:**
- `flutter build ios --release` and `flutter build appbundle` both succeed
- Install on physical iOS + Android device and smoke-test the golden path
- Lighthouse-equivalent checks for privacy + metadata completeness
