## Handoff notes

### 2026-04-22 — implementer

**What was done:**

Audited all seven acceptance criteria against the existing codebase. Six of the seven were already fully implemented:

- Public link (AllowAny, UUID-scoped) — exists in ESigningPublicSignDetailView
- Touch signature capture — SignatureCapture.vue uses signature_pad with touch-none canvas, correct devicePixelRatio scaling, and requestAnimationFrame layout timing
- Signed PDF via Gotenberg — generate_signed_pdf + signed_pdf_file.save() in the submit handler
- ESigningAuditEvent — log_esigning_event captures ip_address (honouring X-Forwarded-For), user_agent, and created_at for every event type
- WebSocket broadcast — _broadcast_ws pushes to esigning_{pk} group on every signer and submission event
- signed_pdf_file stored — generated and saved to esigning/signed_pdfs/ when all signers complete; signed_pdf_hash (SHA-256) also stored

**Bug fixed — resend does not invalidate prior links:**

ESigningResendView and ESigningCreatePublicLinkView both created a new ESigningPublicLink without expiring old active links for the same signer role. This meant multiple valid signing URLs could exist simultaneously, and resending did not actually revoke the old URL.

Fix: Added _expire_prior_links(submission, signer_role) helper in views.py that sets expires_at = now() on all non-expired links for that signer role before creating the new one. Called in both ESigningResendView.post() and ESigningCreatePublicLinkView.post().

New test: ESigningResendTests::test_resend_expires_prior_links in test_hub/esigning/integration/test_esigning.py — creates two prior links, calls resend, asserts both prior links expired and exactly one new active link exists. All 4 resend tests pass (167 total pass).

**Known pre-existing issue (not caused by this change):**
test_rate_limits.py::TestPublicSignMinuteThrottle::test_429_after_threshold_exceeded fails when the full esigning suite runs together due to shared throttle cache state from other test classes. It passes cleanly in isolation (17/17 in test_rate_limits.py). See discovery 2026-04-22-throttle-test-isolation.md.

**Caveats for reviewer:**
- The PDF layout parity criterion (esigning_pdf_layout_compare) is an E2E MCP tool check — the tester should run it against a live Gotenberg instance.
- The name variable was removed from the resend view's unused local (it was set but never used after the _notify_next_signer refactor).

### 2026-04-22 — Review passed

Checked against all seven acceptance criteria. All pass.

1. **Public link without login** — `ESigningPublicSignDetailView` and `ESigningPublicDocumentView` both carry `permission_classes = [AllowAny]` and `authentication_classes = []`. UUID-scoped, correct.

2. **Touch signature capture** — `SignatureCapture.vue` uses `signature_pad`, canvas has `class="w-full touch-none"`, `devicePixelRatio` scaling applied in both `initCanvas()` and `resizeCanvas()`, double-RAF layout guard ensures canvas is measured after paint. Confirmed at `admin/src/views/signing/SignatureCapture.vue:44-49` and `119-161`.

3. **PDF layout parity** — deferred to tester as noted; requires live Gotenberg instance. E2E tool `esigning_pdf_layout_compare` is in the test plan.

4. **ESigningAuditEvent** — `audit.py` captures `ip_address` (XFF → REMOTE_ADDR fallback), `user_agent`, and `created_at` (implicit on model create) for every event. Pre-existing XFF spoofability concern already filed as discovery `2026-04-22-xff-trust-all-apps.md` by a prior reviewer — out of scope here.

5. **WebSocket real-time** — `_broadcast_ws` in `webhooks.py` is called on signer completion and submission completion in `views.py:671,686`. Group name `esigning_{pk}` is consistent.

6. **Resend / link invalidation** — `_expire_prior_links()` is called in both `ESigningResendView.post()` (line 424) and `ESigningCreatePublicLinkView.post()` (line 766). ORM query is parameterised (no raw SQL). `signer_role__iexact` prevents case-mismatch bypass. Test `test_resend_expires_prior_links` covers both prior links expiring and exactly one new active link existing.

7. **signed_pdf_file stored** — generated and saved with SHA-256 hash in the submit handler at `views.py:666-667`.

**Security / POPIA pass:** No new endpoints introduced. The two modified endpoints (`ESigningResendView`, `ESigningCreatePublicLinkView`) retain `permission_classes = [IsAgentOrAdmin]`. No raw SQL, no PII logged, no secrets in diff. `_expire_prior_links` is a module-level private function not exposed to routing.
