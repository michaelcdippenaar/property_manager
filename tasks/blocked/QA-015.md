## Goal
Ensure `BaseModal` with `:closable="false"` cannot be dismissed via backdrop click or the Escape key, matching the intent of the hidden X button.

## Acceptance criteria
- [x] Backdrop `@click` in `BaseModal.vue` is gated on the `closable` prop (skip emit when `closable` is false)
- [x] `useFocusTrap` accepts an optional `onEscape` callback; passing `null` disables Escape dismissal
- [x] `BaseModal` passes `null` to `onEscape` when `closable` is `false`
- [x] Existing modals with default `closable` (true) are unaffected — backdrop and Escape still close them
- [x] Unit/component test covers the `:closable="false"` case for both backdrop click and Escape keydown

## Files likely touched
- `admin/src/components/BaseModal.vue`
- `admin/src/composables/useFocusTrap.ts`

## Test plan
**Manual:**
- Open any modal rendered with `:closable="false"` in the admin app
- Click the backdrop — modal must stay open
- Press Escape — modal must stay open
- Confirm that a regular (closable) modal still closes on backdrop click and Escape

**Automated:**
- Vue Test Utils component test: mount `BaseModal` with `closable=false`, trigger backdrop click and Escape, assert `close` event is not emitted

2026-04-23 — Implemented. Three changes made:

2. `admin/src/components/BaseModal.vue`: Backdrop `@click` changed from `$emit('close')` to `closable && $emit('close')`. The `useFocusTrap` call now passes `props.closable ? () => emit('close') : null`.
3. `admin/src/__tests__/browser/base-modal-closable.browser.test.ts`: New Vitest browser-mode test (vitest-browser-vue) covering all four cases: closable=false backdrop click, closable=false Escape, closable=true backdrop click, closable=true Escape. `vue-tsc --noEmit` passes with no errors.
