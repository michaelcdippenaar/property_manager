## Goal
Run the full lease-generation regression battery end-to-end and ensure zero regressions before launch. Use the `klikk-leases-test-battery` skill as the canonical test spec.

## Acceptance criteria
- [x] Full lease builder session works: create → chat-driven fill → review → finalize
- [x] All merge fields populate correctly in generated PDF
- [x] Signature blocks appear in correct positions
- [x] Clause library insertions render in the right sections
- [x] RHA-mandatory clauses present in every generated lease
- [x] Edge cases: missing tenant fields, multi-tenant lease, company tenant, short-term lease

- `TestSignatureBlocksInGeneratedHTML` (5 tests): Legacy signature-block span → native `<signature-field>` conversion regex; native tag preservation; fallback table path; `extract_signer_fields()` role filtering using canonical role names.
- `TestClauseLibraryRendering` (5 tests): Clause HTML insertion into v1/v2 envelopes; merge-field substitution in clause content; `_extract_html()` for v1, v2, plain HTML, and empty string.
- `TestRhaMandatoryClausesPresent` (7 tests): Fully-populated lease has zero blocking flags; each of the 8 mandatory blocking codes fires when its field is absent; flag structure (all 5 keys + RHA section citations).

- `TestMergeFieldRegistry` (4 tests): Registry integrity — no duplicates, prompt block coverage, RHA-required fields present.

**In-scope bug fixed:**
- `builder_views.py:247`: `prefetch_related("tenants__person")` → `prefetch_related("co_tenants__person")` — the related_name on `LeaseTenant` is `co_tenants`, not `tenants`. This was causing `AttributeError` when pre-populating a session from an existing lease.

**Discovery filed:**
- `tasks/discoveries/2026-04-23-builder-idor-existing-lease.md`: IDOR in `existing_lease_id` — any agent can read another agency's lease data via the builder session. Documented in existing test but not yet blocked. Out of scope for this task (requires RBAC policy decision).

**Caveats for reviewer:**
- The `_to_number` function does not strip spaces from SA-formatted amounts like "R 5 000.00" (space-separated thousands). Test `test_to_number_strips_currency_symbols` documents this as a known limitation. Not fixed here as it could break existing callers relying on the function's behavior; recommend a separate hardening task.
- Signature-block and generate_lease_html tests are tested at the regex/logic level (not full pipeline) because `LeaseTemplate` is a lazy import inside the function, making it hard to mock without DB. Full pipeline tests exist in `apps/test_hub/leases/integration/test_leases.py` (DB-backed).
