## Goal
Narrow `scripts/check_task_board_integrity.sh`'s DEC-NNN reference validity check so it only scans the `depends_on:` frontmatter line — preventing false positives when prose or handoff notes mention a DEC ID as an example or test string.

## Context
Discovered during OPS-025 review: `scripts/check_task_board_integrity.sh:201` runs `grep -oE 'DEC-[0-9]+' "$filepath"` over the entire task file. The OPS-025 AC specifies only `depends_on:` entries should be checked. Writing a handoff note such as "I inserted bogus DEC-9999 to test the guard" will trip the check on the file that documents the guard itself.

## Acceptance criteria
- [x] DEC-reference scanning in `check_task_board_integrity.sh` reads only the `depends_on:` frontmatter field (not the whole file)
- [x] Re-run on current `tasks/` tree exits 0 — no regression in catching genuinely dangling `depends_on` DEC entries
- [x] A handoff note containing the text `DEC-9999` in prose does NOT trip the check
- [x] `docs/ops/task-board-guard.md` updated to reflect the narrowed scope

## Files likely touched
- `scripts/check_task_board_integrity.sh` (line ~201, DEC grep logic)
- `docs/ops/task-board-guard.md` (update description of DEC check)
