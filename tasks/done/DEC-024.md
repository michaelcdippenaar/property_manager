## Question

When an agent activates a lease and mandatory RHA fields (deposit amount, inspection-in date, inspection signed-off) are missing, should the system hard-block activation or show a soft warning?

## Options

**Option A — Hard-block (recommended long-term)**
- Pros: guarantees RHA compliance at all times; protects agency from liability.
- Cons: friction at launch; edge cases (legacy imports, partial data) may block legitimate activations.

**Option B — Soft-warn for launch, hard-block post-launch**
- Pros: unblocks v1.0 launch; first-client dry-run can proceed without data gaps causing blocks; agent acknowledges risk explicitly.
- Cons: non-compliant leases can be activated if agent dismisses the banner.

## Answer

FINAL (2026-04-24 — MC):

- **Launch behaviour: soft-warn**
  - Banner shown on lease activation flow when any of the following are missing: deposit amount, inspection-in date, inspection signed-off.
  - Agent must tick "acknowledged" to proceed past the banner.
  - Lease activation is not blocked — agent can proceed after acknowledgement.
- **Hard-block is a v1.1 item.** A backlog task has been filed to implement the hard-block gate post-launch (see RNT-QUAL-048).

## Downstream impact

- RNT-SEC-007 — already in done/; soft-warn gate was the shipped behaviour. Decision confirms this is correct for launch.
- Any in-progress lease activation tasks: soft-warn banner + "acknowledged" tick is the launch acceptance criterion.
- RNT-QUAL-048 — new backlog task filed for v1.1 hard-block gate.
