## Question
Which cloud provider, region, and compute shape do we target for the v1.0 production environment?

## Options (summarised)
- **AWS af-south-1 (Cape Town)** — POPIA data residency, IAM-aligned with DEC-002 (SES) and DEC-003 (SSM). Recommended.
- **Hetzner Falkenstein** — cheapest raw compute, EU data residency (GDPR-only, not POPIA af-south-1).
- **DigitalOcean** — simpler UX, no af-south-1 region.

## Answer

FINAL (2026-04-24 — MC):

- **Cloud:** AWS
- **Region:** af-south-1 (Cape Town) — chosen for POPIA data residency
- **Compute shape (default, revisable later):**
  - EC2 t3.medium — Django + Caddy
  - RDS Postgres db.t4g.small
  - ElastiCache Redis t4g.micro
  - S3 for media
  - SES for transactional email (aligns with DEC-002)
  - CloudFront optional post-launch
- **SSM Parameter Store af-south-1** for secrets (aligns with DEC-003)

## Downstream impact

- **OPS-001** — staging CI/CD already operational. Production deploy target confirmed: AWS af-south-1, EC2 t3.medium, Docker + Caddy. Update `docs/ops/ci-cd.md` to reflect prod target when production deploy workflow is added.
- **OPS-002** — Sentry projects should use region-nearest ingestion. DSN config unchanged by this decision; hosting is a separate concern.
- **OPS-003** — S3 backup target confirmed: `klikk-backups-af` in `af-south-1`. Provision bucket, IAM user, lifecycle rules per `docs/ops/backup-restore.md`. Unblocked.
- **OPS-006** — DNS/TLS/CDN config targets AWS AF. CloudFront is the CDN option for post-launch; use Caddy + Let's Encrypt at launch. All subdomains per DEC-013.
- **OPS-009** — Secrets vault confirmed: AWS SSM Parameter Store af-south-1 (aligns with DEC-003 recommendation). OPS-009 AC item 1 is now resolved: SSM is the choice.
