## Goal
Move all `--cov` flags into `pytest.ini` as the single source of truth so local and CI runs use identical coverage config and the maintenance trap of split configuration is eliminated.

## Acceptance criteria
- [x] `pytest.ini` `addopts` includes `--cov=apps --cov-report=term-missing --cov-report=html:htmlcov --cov-report=xml:coverage.xml`
- [x] CI `pytest` step in `.github/workflows/ci.yml` is plain `pytest --tb=short -q` with no `--cov` flags
- [x] CI XML coverage report still uploaded to coverage service (confirm `coverage.xml` artifact path unchanged)
- [x] Local `pytest` run produces HTML and XML reports

## Files likely touched
- `backend/pytest.ini`
- `.github/workflows/ci.yml`
