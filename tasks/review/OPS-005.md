## Goal
Replace every plaintext / default-styled email with a branded, mobile-friendly, accessible HTML template — content sourced from `content/emails/` so copy edits don't require code changes.

## Acceptance criteria
- [x] Template engine: MJML or maizzle → responsive HTML compiled at build
- [x] Base layout: Klikk brand (Navy #2B2D6E, Accent #FF3D7F, Bricolage Grotesque fallback → system serif; DM Sans fallback → system sans), footer with unsubscribe + info officer contact
- [x] Templates for: welcome (agent + tenant), mandate sent, mandate signed, lease sent for signing, lease signed, rent reminder, rent received, rent overdue, maintenance update, invite tenant, password reset, 2FA enrollment, 2FA recovery code reissue
- [x] Plaintext fallback for every HTML
- [x] Content sourced from `content/emails/*.md` (markdown + frontmatter)
- [x] Accessibility: alt text on images, sufficient contrast, no text-in-image
- [x] Deliverability: SPF + DKIM + DMARC set on sender domain; warmup schedule documented

## Files likely touched
- `content/emails/*.md` (new directory)
- `backend/apps/notifications/email_templates/*.mjml`
- `backend/apps/notifications/services/email.py`
- `docs/ops/email-deliverability.md`

## Test plan
**Manual:**
- Send each template type to a real inbox across Gmail / Outlook / Apple Mail → renders correctly, links work
- Mail-tester.com score ≥ 9/10

## Handoff notes

### 2026-04-22 — implementer

**What was built:**

- `backend/apps/notifications/email_templates/base.mjml` — single MJML source with Django template variables (`{{ heading }}`, `{{ body_text }}`, `{{ cta_url }}`, `{{ note }}`, etc.)
- `backend/apps/notifications/email_templates/compiled/base.html` — pre-compiled responsive HTML (compiled at build via `npx mjml base.mjml -o compiled/base.html`). Django `{% if cta_url %}` and `{% if note %}` conditionals added post-compilation to hide optional rows cleanly.
- `content/emails/*.md` — 14 markdown files (one per template type), each with YAML frontmatter (`subject`, `heading`, `cta_label`, `cta_url_key`, `note`) and plain body copy. Content is the single source of truth — copy changes require no code changes.
- `backend/apps/notifications/services/email.py` — `render_template_email(template_id, context)` and `send_template_email(template_id, to_emails, context)`. Reads `.md`, renders body as minimal markdown-to-HTML, merges into compiled base template via Django template engine, generates plaintext fallback.
- `backend/apps/notifications/services/__init__.py` — the old `services.py` was promoted to a package `__init__.py`. All existing callers (`from apps.notifications.services import send_email`) continue to work unchanged. One relative import `.models` was changed to an absolute import `apps.notifications.models` to satisfy Python's package resolution.
- `backend/apps/accounts/views.py` — password reset call updated to use `send_template_email("password_reset", ...)` with full branded HTML.
- `docs/ops/email-deliverability.md` — SPF/DKIM/DMARC DNS record examples, 7-week warmup schedule, ESP recommendation (Amazon SES `af-south-1`), Mailpit local testing instructions, bounce/complaint monitoring thresholds, POPIA compliance notes.

**Smoke check:** all 14 template IDs render without error, produce correct subjects (including `{{ var }}` interpolation), correct CTA URLs, branded HTML with Klikk footer, and plaintext fallbacks.

**Caveat for reviewer:** The `{% if cta_url %}` and `{% if note %}` conditionals in `compiled/base.html` were hand-inserted post-MJML-compilation. If `base.mjml` is ever recompiled, those two blocks must be re-patched in the compiled output. A build script (`scripts/compile_mjml.sh` or Makefile target) should be added in a follow-up task to automate this. See discovery note below.

**Discovery filed:** `tasks/discoveries/2026-04-22-mjml-build-script.md` — tracks the need for a Makefile/script target that compiles MJML and re-inserts the Django conditional blocks automatically.
