## Goal
Surface a "How to pay rent" section in the tenant Lease tab showing banking details, payment reference, and due date so tenants never need to call the agent to find out where to pay.

## Acceptance criteria
- [x] Tenant Lease tab shows a "How to pay" section with: bank name, account number, branch code, account holder, and unique payment reference
- [x] Payment reference is displayed in a copyable monospace field (tap-to-copy on mobile)
- [x] Due date is surfaced alongside the payment section (re-use existing field if already shown)
- [x] Section includes a plain-language note on the consequence of late payment per RHA (e.g. "Payment received after the [due day] of the month may be subject to a late penalty as per your lease agreement")
- [x] Tenant serializer/endpoint exposes payment details scoped to the authenticated tenant's lease only — no cross-tenant data leakage
- [x] `BankAccount` is resolved from the lease's property/landlord and serialized; `payment_reference` from `Lease.payment_reference`
- [x] Section hidden gracefully if no `BankAccount` is on record (with placeholder: "Your agent will provide payment details")

## Files likely touched
- `tenant/src/views/leases/LeaseView.vue` — add "How to pay" section
- `backend/apps/leases/schema.py` — extend tenant lease serializer to include payment details (line ~109)
- `backend/apps/leases/views.py` — tenant lease detail endpoint
- `backend/apps/properties/models.py` — `BankAccount` (lines 222-229)

## Handoff notes
2026-04-23 rentals-pm: Promoted from discovery `2026-04-23-tenant-no-rent-payment-instructions.md` (found during UX-002 audit). P1 — high-frequency tenant query; every active tenant is affected monthly. Serializer change must be reviewed for data-scoping correctness.

2026-04-23 implementer: Implemented end-to-end.

**Backend (`backend/apps/leases/serializers.py`):**
Added `payment_details = SerializerMethodField()` to `LeaseSerializer`. `get_payment_details()` delegates to the existing `_get_landlord_info()` from `apps.esigning.services` (same logic already used for merge fields and PDF generation) to resolve `BankAccount` via `Lease → Unit → Property → PropertyOwnership(is_current) → Landlord.bank_accounts`. When no bank account is on record, `bank_account` key is `None` so the frontend can show a graceful placeholder. `payment_reference` and `rent_due_day` are always returned. No changes to views.py or models.py were needed — data scoping is already enforced at queryset level in `LeaseViewSet.get_queryset()` which filters to the authenticated tenant's leases only.

**Frontend (`tenant/src/views/leases/LeaseView.vue`):**
Added "How to pay" section between "Lease Details" and "Calendar". Shows: bank name, account number (monospace), branch code (monospace), account holder, copyable payment reference row (tap-to-copy with "Reference copied" toast), and due date with RHA late-payment notice. Graceful empty state when `payment_details.bank_account` is null. New Lucide icons imported: `Landmark`, `Hash`, `Code2`, `User`, `Receipt`, `Copy`, `CalendarClock`. Added `fade-up` transition CSS for the copy toast.

**Tests (`backend/apps/leases/tests/test_tenant_payment_details.py`):**
16 unit tests, all passing (no DB required, mocks only). Covers: all bank fields populated, `bank_account: null` when missing, `_get_landlord_info` returning None, `payment_reference` and `rent_due_day` always present, `payment_details` in serializer output, field is read-only.

**Caveats for reviewer:**
- The `payment_details` field is appended to the existing `LeaseSerializer` which uses `fields = "__all__"`. It will appear in ALL lease API responses (agent/admin portal too), not just tenant-facing ones. This is a minor scope increase but not a data-leakage risk since the serializer's queryset scoping is the gate. Reviewer may want to confirm this is acceptable or split into a tenant-specific serializer.
- `schema.py` (GraphQL) was not touched — it already exposes `payment_reference` directly on `LeaseType`. The new banking details are not surfaced there, but that is only used in internal tooling.
