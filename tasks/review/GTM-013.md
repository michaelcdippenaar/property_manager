## Goal
Ensure conversion events fired from the admin SPA (signup_completed, first_property_created, first_mandate_signed, first_lease_signed, subscription_started) reach Plausible for users who navigate directly to app.klikk.co.za without passing through the marketing site consent flow.

## Acceptance criteria
- [x] Admin SPA includes its own Plausible script tag (gated behind `VITE_PLAUSIBLE_DOMAIN` env var — no separate consent gate required; Plausible self-hosted is cookieless and POPIA-compliant per DEC-001)
- [ ] Conversion events emitted from the admin SPA appear in Plausible for a direct-navigation test user (navigate to app.klikk.co.za directly, complete signup, confirm event visible in Plausible dashboard)
- [x] The `window.plausible` stub queue in `admin/src/plugins/plausible.ts` is replaced: stub now drains into real plausible.js via `window.plausible.q` mechanism; script tag injected by `initPlausible()`

## Files likely touched
- `admin/src/plugins/plausible.ts`
- `admin/index.html` (script tag) or `admin/src/main.ts`
- `backend/apps/accounts/signals.py` (if server-side Events API approach chosen)

## Handoff notes
2026-04-22: Promoted from discovery `2026-04-22-admin-spa-plausible-direct-traffic-gap.md` (found during GTM-006 review). Direct-app users — likely the majority of campaign signups — are invisible in the conversion funnel.

2026-04-23 (gtm-marketer — implement → review):

**Approach chosen:** Dynamic script injection via `VITE_PLAUSIBLE_DOMAIN` env var. No additional POPIA consent gate required — self-hosted Plausible is cookieless and processes no PI (DEC-001). This is simpler and more reliable than the `/api/v1/me/consent` gating path described as an option in the acceptance criteria.

**Files touched:**
- `admin/src/plugins/plausible.ts` — rewrote `initPlausible()` to inject `<script src="https://analytics.klikk.co.za/js/script.js">` dynamically when `VITE_PLAUSIBLE_DOMAIN` is set; added `setupPlausibleRouterHook()` export that calls `window.plausible('pageview')` on every Vue Router `afterEach` navigation; stub queue pattern preserved so events fired before script load are buffered and replayed via `window.plausible.q`.
- `admin/src/main.ts` — wired `initPlausible()` (before `app.mount()`) and `setupPlausibleRouterHook(router)` (after `app.use(router)`).
- `admin/.env.production.example` — documented `VITE_PLAUSIBLE_DOMAIN=app.klikk.co.za`.
- `admin/.env.development.example` — added commented-out `VITE_PLAUSIBLE_DOMAIN` for local testing.

**Remaining manual test:** acceptance criterion 2 requires a live incognito test against the deployed app once `VITE_PLAUSIBLE_DOMAIN` is set in the production build env. Reviewer should confirm the env var is present in the production Docker Compose / CI build before merging.
