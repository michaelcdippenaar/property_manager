## Handoff notes

### 2026-04-22 — implementer

**Changes shipped:**

1. **Copy audit (pass)** — all features shown on homepage `features` section are BUILT. Three BI features (`market_analytics`, `agency_dashboard`, `lead_analytics`) are PLANNED and show a "Coming Soon" badge — correctly labeled, not falsely advertised as live.

2. **SEO / structured data** — Added `@graph` JSON-LD block to `BaseLayout.astro` with `Organization`, `SoftwareApplication`, and `LocalBusiness` (SA) types. `canonical`, OG, and Twitter card tags were already present. `sitemap-index.xml` was already generated by `@astrojs/sitemap`.

3. **`robots.txt`** — Created `website/public/robots.txt` with `Allow: /` and sitemap pointer. Verified it copies to `dist/robots.txt` on build.

4. **Contact form email** — Default was `mc@klikk.co.za`; changed to `hello@klikk.co.za`. Overridable via `PUBLIC_CONTACT_EMAIL` env var. POPIA consent checkbox was already present in the form.

5. **POPIA consent banner** — New component `website/src/components/ConsentBanner.astro`. Sets `klikk_consent` cookie (1 year, `SameSite=Lax; Secure`). Two actions: "Accept analytics" (value: `analytics`) and "Essential only" (value: `essential`). Fires 1.2 s after page load. Skips display if cookie already set. Included in `BaseLayout.astro`.

6. **Dead links removed** — Footer had `/blog` and `/about` links that pointed to non-existent pages. Removed both. Remaining footer links (`/#features`, `/pricing`, `/contact`, `/legal/privacy`, `/legal/terms`) all resolve to built pages.

7. **Build fix (in-scope sub-issue)** — Pre-existing build blocker in `astro.config.mjs`: `manualChunks: { 'vendor-vue': ['vue'] }` causes Rollup error because `@astrojs/vue` externalises Vue. Removed the `manualChunks` block. Build now completes cleanly (7 pages).
