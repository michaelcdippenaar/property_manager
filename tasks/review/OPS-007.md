## Goal
`content/product/pricing.yaml` defines tiers, but nothing enforces them. Wire tier → feature-flag so agencies see + can do only what their tier allows.

## Acceptance criteria
- [x] `Agency.subscription_tier` FK to a tier from pricing.yaml (synced at startup)
- [x] `agency.has_feature("X")` helper reads tier entitlements
- [x] Permission decorator `@requires_feature("ai_lease_generation")` on relevant views
- [x] Admin SPA: gated features show "Upgrade" CTA instead of 403
- [x] Quota enforcement: per-tier property count, per-tier owner/tenant seat count, per-tier AI chat turns per month — each with soft warning at 80%, hard block at 100% (configurable)
- [x] Usage metrics visible on Agency Settings → Billing tab
- [x] Upgrade / downgrade workflow (v1.0: manual Stripe link; self-serve billing deferred if not ready)
