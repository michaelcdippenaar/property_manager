### 2026-04-22 — reviewer: changes requested

Two fixes required before this can move to testing.

**1. Fail-open path in `@requires_feature` decorator — `backend/apps/accounts/decorators.py` lines 65–87**

When the decorator cannot identify the DRF Request (the `else: request = None` branch at line 67), it falls through silently and calls the view, granting access without any feature check. Although the current gated views all carry `permission_classes = [IsAgentOrAdmin]` which prevents unauthenticated access, the decorator's own contract is to enforce the tier gate and must not silently skip it. A non-standard CBV layout (e.g. a method called with an extra `format` kwarg, or a view that wraps `self` differently) would bypass the gate entirely.

Fix: when `request is None`, either raise `ImproperlyConfigured` in debug mode or return a `Response({"detail": "Feature enforcement failed."}, status=500)` rather than passing through. At minimum, add an `else: raise ImproperlyConfigured(f"@requires_feature could not find DRF Request for {feature_slug}")` so the wiring error is caught immediately in development.

**2. AI contract quota counts by `updated_at` not by year of finalization — `backend/apps/accounts/tier_service.py` line 248**

`_ai_contracts_this_year()` filters `status="finalized", updated_at__year=year`. `updated_at` is an `auto_now=True` field that changes on every model save after finalization (re-download, metadata patch, admin touch, etc.). A lease finalized in December 2026 that receives any update in January 2027 will no longer be counted against the 2026 quota, freeing a slot incorrectly. Conversely, a lease created and finalized in 2025 that is touched in 2026 will wrongly consume a 2026 quota slot.

Fix: filter on `created_at__year=year` (the session's creation year) or introduce a `finalized_at = models.DateTimeField(null=True)` field on `LeaseBuilderSession` set at finalization time, and filter on `finalized_at__year=year`. Using `created_at` is the simplest correct fix and requires no model change.

Discovery filed: `tasks/discoveries/2026-04-22-requires-feature-request-detection.md` (non-standard CBV wrapping as a broader hardening concern).

### 2026-04-22 — implementer (reviewer fixes)

**Fix 1 — `@requires_feature` fail-open removed (`decorators.py`):**
- Added `from django.core.exceptions import ImproperlyConfigured` import.
- Replaced `request = None` / silent fall-through with a hard fail: in DEBUG mode raises `ImproperlyConfigured` with a descriptive message; in production returns `HTTP 500 {"detail": "Feature enforcement failed."}`. The view is never called without a successful tier check.
- The `if request is not None and …` guard was simplified to `if hasattr(request, 'user') and …` since `request` is now always a valid DRF Request at that point (or we have already returned).

**Fix 2 — AI contract quota year-bucket uses `created_at` (`tier_service.py`):**
- Changed `updated_at__year=year` → `created_at__year=year` in `_ai_contracts_this_year()`.
- `updated_at` is `auto_now=True` and shifts the bucket on every save; `created_at` is immutable and correctly anchors the session to the year it was started.
