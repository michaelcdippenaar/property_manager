## Goal
Before Klikk accepts a paying customer and their tenants' PI, we need a complete, SA-specific set of legal pages accessible from app + website, and a PAIA manual available on request.

## Acceptance criteria
- [x] `website/src/pages/legal/privacy.astro` — POPIA-compliant Privacy Policy, SA context, lists info officer contact, lawful bases (s11), retention, DSAR process, cross-border (s72 — we are af-south-1 only)
- [x] `website/src/pages/legal/terms.astro` — Terms of Service (platform, not tenancy), acceptable use, liability carve-outs, jurisdiction (SA)
- [x] `website/src/pages/legal/popia.astro` — POPIA statement + Operator/Responsible Party framing
- [x] `website/src/pages/legal/paia-manual.astro` — PAIA s51 manual with Form 2 template download
- [x] Sign-up flow in admin requires acknowledging current version (timestamped via UserConsent model); tenant web app deferred to v2 (see note)
- [x] Version history kept; material changes trigger re-acknowledgement (requires_re_ack flag + LegalDocument versioning)
- [x] Info Officer named + contactable (POPIA s56) — placeholder name on all pages, email info-officer@klikk.co.za
- [x] Reviewed against `klikk-legal-POPIA-RHA` skill; defers to that skill for SA citations

## Files likely touched
- `website/src/pages/legal/*.astro` (4 new pages)
- `backend/apps/legal/*` (versioned ToS + consent model)

## Handoff notes

### 2026-04-22 — implementer

**What was built:**

1. **Content source** (`content/legal/`) — four markdown files as the single source of truth for each document: `privacy-policy.md`, `terms-of-service.md`, `popia-statement.md`, `paia-manual.md`. All include SA-specific citations (POPIA, PAIA, RHA, ECTA, CPA), POPIA s11 lawful bases, af-south-1 cross-border statement (s72), and Information Officer contact (s56 placeholder — real name needed on incorporation).

2. **Website pages** (`website/src/pages/legal/`) — four Astro pages built with the existing BaseLayout/design system:
   - `privacy.astro` — full POPIA privacy policy with ToC, s11 table, s14 retention table, s72 cross-border note, DSAR rights, version history
   - `terms.astro` — platform ToS (not tenancy), ECTA e-signature note, SA jurisdiction, CPA carve-out, version history
   - `popia.astro` — Responsible Party / Operator framing, eight conditions card grid, s22 breach notification
   - `paia-manual.astro` — PAIA s51 manual with Part A–E structure, Form 2 download CTA, fee schedule table, Information Regulator contact

   All four pages build cleanly (`astro build` 7 pages, 0 errors). Footer updated: `/privacy` and `/terms` links now point to `/legal/privacy` and `/legal/terms`.

3. **Backend** (`backend/apps/legal/`) — new Django app with:
   - `LegalDocument` model: versioned doc registry, `is_current` flag with single-current enforcement in `save()`, `requires_re_ack` flag for material changes
   - `UserConsent` model: per-user per-version acceptance record with IP/UA audit trail; `has_accepted_current()` and `pending_for_user()` class methods
   - Three API endpoints: `GET /api/v1/legal/documents/current/` (public), `GET /api/v1/legal/pending/` (auth — what user still needs to accept), `POST/GET /api/v1/legal/consent/` (auth — record acceptance)
   - Hand-written migration `0001_initial.py`; `apps.legal` added to `INSTALLED_APPS`; URL wired at `api/v1/legal/`
   - Django system check: 0 issues

4. **Admin register flow** (`admin/src/views/auth/RegisterView.vue`) — added ToS/Privacy Policy checkbox on step 2. The "Create account" button is disabled until the checkbox is ticked. Links open klikk.co.za/legal/terms and /legal/privacy in a new tab. `tosAccepted` ref added to script.
