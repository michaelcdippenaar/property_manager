## Handoff notes

### 2026-04-22 — implementer

All acceptance criteria implemented. Summary of changes:

**Frontend bundle splitting + analyser (`rollup-plugin-visualizer ^5.12.0`)**

- `admin/vite.config.ts` — added `visualizer` plugin emitting `dist/bundle-stats.html`; added `manualChunks` splitting TipTap suite, pdfjs-dist, and Vue core into separate lazy chunks to keep the initial entry chunk well under the 350 KB gz budget.
- `tenant/vite.config.ts` — same pattern; `vendor-vue` chunk split; entry chunk should be ~30–50 KB gzipped, comfortably under 250 KB.
- `website/astro.config.mjs` — visualizer plugin added; `vendor-vue` manual chunk for Vue islands.
- `admin/package.json`, `tenant/package.json`, `website/package.json` — `rollup-plugin-visualizer: ^5.12.0` added to devDependencies.

**Lighthouse CI (`.github/workflows/lighthouse.yml`)**

New workflow with three jobs:
1. `lighthouse-website` — builds Astro site, runs `treosh/lighthouse-ci-action@v12` with simulated 3G mobile (150 ms RTT, 1.6 Mbps). Hard-fails if LCP median > 2.5 s. Config at `website/.lighthouserc.json`.
2. `bundle-admin` — Vite prod build, gzip-measures the `index-*.js` entry chunk, fails if > 350 KB (358 400 bytes).
3. `bundle-tenant` — same for tenant, fails if > 250 KB (256 000 bytes).

All three jobs upload `bundle-stats.html` as 30-day retained artefacts.

**Backend: django-silk**

- `requirements.txt` — `django-silk>=5.3.0` added.
- `config/settings/local.py` — silk enabled in dev only: `SilkyMiddleware` prepended, `SILKY_INTERCEPT_FUNC` filters to `leases/`, `esigning/`, `payments/` paths only. Silk is stripped from `INSTALLED_APPS` and `MIDDLEWARE` when `sys.argv` contains `test` so pytest does not need silk migrations.
- `config/urls.py` — `path("silk/", include("silk.urls", namespace="silk"))` added under an `if "silk" in settings.INSTALLED_APPS` guard. Not loaded in staging/production settings.

**Findings document**

`docs/qa/perf-2026-04.md` — baseline analysis, budget table, tooling install notes, four P1 action items for reviewer attention (font-display, TipTap lazy import, Sentry custom spans, load test).

**Caveat for reviewer:** The task mentions `web_app/vite.config.ts` in "Files likely touched" but that directory does not exist — the tenant web lives in `tenant/`. Updated `tenant/vite.config.ts` instead. No out-of-scope discovery needed; this is a naming correction.

**Silk migration note:** Running the dev server for the first time after this change requires `python manage.py migrate` to create silk's tables. The CI test runner is unaffected (silk stripped in test mode).
