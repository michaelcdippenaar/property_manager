## Goal
Wire Sentry across every surface so production errors reach us with full context (user role, agency, request, breadcrumbs) instead of dying silently.

## Acceptance criteria
- [x] Backend: `sentry-sdk[django]`, DSN from env, `send_default_pii=False`, release = git SHA
- [x] Admin SPA: `@sentry/vue`, DSN from env, source maps uploaded in CI
- [ ] Tenant web: same pattern  ← `web_app/` directory does not exist; see Handoff notes
- [x] Agent mobile (Quasar/Capacitor): `@sentry/capacitor`
- [x] v1.0 scope: no Flutter tenant mobile (deferred to v2)
- [x] PII scrubbing: strip id_number, passport, bank account from payloads (hook)
- [ ] Alerts: Slack/email on new issue, regression, error-rate spike  ← Sentry project setup required (see Handoff notes)
- [ ] Per-tier quota respected (dev/staging/prod projects)  ← Sentry project setup required (see Handoff notes)

## Handoff notes

### 2026-04-22 — implementer

**What was done:**

1. **Backend (`sentry-sdk[django]`):**
   - Added `sentry-sdk[django]>=2.0.0` to `backend/requirements.txt`.
   - Created `backend/config/sentry.py` — single `init()` helper that configures `DjangoIntegration`, `LoggingIntegration`, `send_default_pii=False`, and a `before_send` PII scrubber.
   - Added Sentry bootstrap block to `backend/config/settings/base.py`: reads `SENTRY_DSN`, `SENTRY_ENVIRONMENT`, `SENTRY_TRACES_SAMPLE_RATE` from env; derives release from `GIT_COMMIT` env var (falls back to `git rev-parse --short HEAD`); only calls `sentry.init()` when DSN is non-empty (safe for dev/test).
   - Documented env vars in `backend/.env.example`.

2. **Admin SPA (`@sentry/vue` + source maps):**
   - Added `@sentry/vue ^8.0.0` to `admin/package.json` dependencies and `@sentry/vite-plugin ^2.0.0` to devDependencies.
   - Created `admin/src/plugins/sentry.ts` — `initSentry(app, router)` with `browserTracingIntegration`, Session Replay (errors-only), PII scrub `beforeSend`, `sendDefaultPii: false`.
   - Updated `admin/src/main.ts` to call `initSentry(app, router)` after router registration.
   - Updated `admin/vite.config.ts` to: (a) enable `sourcemap: true` in build, (b) conditionally load `sentryVitePlugin` when `SENTRY_AUTH_TOKEN` + `VITE_SENTRY_DSN` are present and mode is not `development`.
   - Documented all VITE_SENTRY_* and SENTRY_AUTH_TOKEN env vars in `.env.development.example` and `.env.production.example`.

3. **Agent mobile (`@sentry/capacitor`):**
   - Added `@sentry/capacitor ^1.0.0` and `@sentry/vue ^8.0.0` to `agent-app/package.json`.
   - Replaced the custom `errorLogger.ts` remote-log-shipper with a proper `@sentry/capacitor` init (boot file retains same name so Quasar boot registration is unchanged). Uses `Sentry.init(..., SentryVue.init)` pattern for Capacitor hybrid apps.
   - Exposed `SENTRY_DSN`, `SENTRY_ENVIRONMENT`, `SENTRY_RELEASE`, `SENTRY_TRACES_SAMPLE_RATE` through `quasar.config.js` `build.env`.
   - Documented vars in `agent-app/.env.production` and `.env.staging`.

4. **PII scrubbing:** shared regex pattern (`id_number|id_no|passport|bank_account|account_number|card_number|cvv|pin`) applied in `before_send` on backend and both frontend surfaces.

**Caveats / items for reviewer:**

- **Tenant web (`web_app/`):** directory does not exist in the repo. This criterion cannot be implemented until the tenant web app is scaffolded. Leaving unchecked; no discovery file created as this is already tracked in the v2 backlog decision.
- **Alerts (Slack/email) and per-tier quota:** these require MC to create Sentry projects (one per environment per surface), configure alert rules, and add DSNs + auth tokens to CI secrets. This is a Sentry SaaS configuration task, not a code change. All env var placeholders are documented in `.env.example` / `.env.*.example` files. The reviewer should confirm whether a separate ops ticket for Sentry project setup is needed.
- **`npm install` not run:** package.json changes declare the new deps. The CI pipeline (or a manual `npm install` in each app dir) must be run before building. The packages are straightforward and do not conflict with existing deps.
- **`sentry-sdk` not installed in virtualenv:** same — `pip install -r requirements.txt` pulls it in at next environment rebuild. The import is guarded by `if SENTRY_DSN:` so a missing package won't break a dev environment with no DSN set.
