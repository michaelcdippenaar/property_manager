## Goal
Implement the Django backend for the AI Guide widget so the admin frontend can replace its in-process mock intent mapper with real Anthropic claude-3-5-haiku tool-use calls, with persistence and role-scoped action allowlisting.

## Acceptance criteria
- [x] `backend/apps/ai/guide_views.py` — POST `/api/v1/ai/guide/` accepts `{message, portal}`, calls Anthropic claude-3-5-haiku with tool-use, returns `{reply, action}` structured payload
- [x] `backend/apps/ai/guide_tools.py` — allowlisted actions (`create_property`, `list_leases`, `go_to_dashboard`, `view_maintenance`, etc.) scoped by portal role
- [x] `backend/apps/ai/models.py` — `GuideInteraction` model: `user`, `portal`, `intent`, `action_taken`, `completed`, `created_at`
- [x] `backend/config/settings/base.py` — `ANTHROPIC_API_KEY` and `ENABLE_AI_GUIDE` env vars documented
- [x] `backend/config/urls.py` — `/api/v1/ai/guide/` wired (via `apps/ai/urls.py`)
- [x] Frontend mock swapped: `_mockIntentMapper` in `admin/src/stores/aiGuide.ts` replaced with `api.post('/ai/guide/', ...)` (comment stub already in place)
- [x] Unit tests in `backend/apps/ai/tests/test_guide.py` cover: valid intent, unknown intent, role-scoped action blocked

## Handoff notes
2026-04-23: Promoted from discovery `2026-04-23-ai-guide-backend-endpoint.md` (found during RNT-020). RNT-020 was frontend-only sprint; mock stub at `stores/aiGuide.ts` `sendMessage()` with comment "When backend ships, swap this for...". This task delivers the backend and wires it up.

2026-04-23 (implementer): All acceptance criteria delivered.

**What was done:**
- `backend/apps/ai/guide_tools.py` — 10 tool schemas (Anthropic format), `PORTAL_TOOLS` allowlist per portal (agent/owner/supplier), `TOOL_ACTION_MAP` mapping tool names to GuideAction-compatible dicts, `get_tools_for_portal()` and `is_tool_allowed()` helpers.
- `backend/apps/ai/guide_views.py` — `AIGuideView` (POST), Anthropic haiku tool-use loop, safety double-check that rejects disallowed tools even if model hallucinates them, graceful Anthropic API error handling, persists `GuideInteraction` on every request.
- `backend/apps/ai/models.py` — `GuideInteraction` model added (fields: user, portal, message, intent, action_taken, completed, created_at).
- `backend/apps/ai/migrations/0004_add_guide_interaction.py` — auto-generated migration.
- `backend/apps/ai/urls.py` — `guide/` path prepended.
- `backend/config/settings/base.py` — `ENABLE_AI_GUIDE` env var added (default True).
- `admin/src/stores/aiGuide.ts` — `_mockIntentMapper` retained for reference but `sendMessage()` now calls `api.post('/ai/guide/', ...)`. Import for `api` added.
- `backend/apps/ai/tests/test_guide.py` — 6 tests, all passing: valid_intent, unknown_intent, role_scoped_blocked, unauthenticated_rejected, missing_message_400, interaction_persisted.

**Caveats:**
- The `_mockIntentMapper` function body is still present in the store as dead code. Reviewer may choose to remove it in a follow-up cleanup, but leaving it now to avoid scope creep.
- `ANTHROPIC_API_KEY` was already configured in base.py before this task — no change needed beyond documenting `ENABLE_AI_GUIDE`.
- Rate limit on `AIGuideView` is 20/min (UserRateThrottle). Adjust in settings `DEFAULT_THROTTLE_RATES` if needed.
