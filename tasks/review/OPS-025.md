## Goal
A pre-commit hook or CI check that fails when the same task ID (e.g. `RNT-QUAL-006`) exists in more than one `tasks/<state>/` directory simultaneously — preventing stale orphan copies from making the board state untrustworthy.

## Context
`tasks/review/RNT-QUAL-006.md` was accidentally re-created by an unrelated commit (`ce2792aa`) while the authoritative `tasks/done/RNT-QUAL-006.md` already existed. This caused a duplicate board entry. The orphan was caught manually; a guard would catch it automatically.

## Acceptance criteria
- [x] A script (bash or Python) enumerates all `tasks/*/` directories (backlog, in-progress, review, testing, blocked, done), extracts the `id:` frontmatter field from each `.md` file, and errors if any ID appears in more than one directory
- [x] The check is wired into CI (`.github/workflows/ci.yml` — new job or step, runs on every PR)
- [x] The check is also wired as a pre-commit hook (`.pre-commit-config.yaml`)
- [x] An audit of the current `tasks/` tree at the time of implementation confirms zero existing duplicates (or removes them)
- [x] `docs/ops/task-board-guard.md` documents the check and how to resolve a duplicate if detected

## Files likely touched
- `.github/workflows/ci.yml` (new job: `task-board-guard`)
- `.pre-commit-config.yaml` (new local hook)
- `scripts/check_task_duplicates.sh` or `scripts/check_task_duplicates.py` (new)
- `docs/ops/task-board-guard.md` (new)

## Test plan
**Manual:**
- Copy a done task file into backlog/ → script fails with clear error message
- Remove the copy → script passes

**Automated:**
- CI job green on the PR that introduces this check (no current duplicates)
