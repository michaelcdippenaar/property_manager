## Goal
Write production-ready scripts for two first-run welcome videos: one for the agent app (~60-90s), one for the tenant app (~45-60s). Videos shown on first app launch, before user hits dashboard.

## Acceptance criteria
- [x] `content/onboarding/videos/agent-welcome.md` — scene-by-scene, voice-over, on-screen text, visuals, captions
- [x] `content/onboarding/videos/tenant-welcome.md` — same structure
- [x] Each video covers: who Klikk is, what to expect in first 5 minutes, one clear next action
- [x] Accessibility: caption-ready, no audio-only information
- [x] SA English, warm tone, no corporate jargon

## Files likely touched
- `content/onboarding/videos/agent-welcome.md` (new)
- `content/onboarding/videos/tenant-welcome.md` (new)

## Test plan
**Manual review:**
- Script reads naturally aloud within duration
- Each scene has a single clear message
- First-time agent would understand without prior context

## Handoff notes

### 2026-04-23 — ux-onboarding agent

Both scripts written and ready for production review.

**Files created:**
- `content/onboarding/videos/agent-welcome.md` — 6 scenes, 88–92 s runtime, SA-English
- `content/onboarding/videos/tenant-welcome.md` — 5 scenes, 58–62 s runtime, SA-English

**Design decisions:**
- Agent video (90 s): leads with the 4-app platform diagram, then the 15-stage lifecycle ring
  (built stages in Accent pink, PLANNED in grey with "Coming Soon" label — no false advertising),
  then a live Dashboard recording, then the Properties add flow as the single CTA.
- Tenant video (60 s): three-feature focus only — lease access, maintenance logging, AI chat.
  No mention of PLANNED features (screening, deposit, inspections). Tone is deliberately calmer
  than the agent video — new tenants are often anxious.
- Both scripts include a full Shot List table and Accessibility Checklist for the production team.
- Lifecycle ring animation in agent Scene 3 intentionally shows PLANNED stages as grey with text
  label "Coming Soon" so colour-blind viewers still get the distinction.
- No code files touched. Scope constraint (marketing/ and content/ only) respected throughout.
