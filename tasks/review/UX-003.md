## Goal
Write production-ready scripts for two first-run welcome videos: one for the agent app (~60-90s), one for the tenant app (~45-60s). Videos shown on first app launch, before user hits dashboard.

## Acceptance criteria
- [x] `content/onboarding/videos/agent-welcome.md` — scene-by-scene, voice-over, on-screen text, visuals, captions
- [x] `content/onboarding/videos/tenant-welcome.md` — same structure
- [x] Each video covers: who Klikk is, what to expect in first 5 minutes, one clear next action
- [x] Accessibility: caption-ready, no audio-only information
- [x] SA English, warm tone, no corporate jargon

## Files likely touched
- `content/onboarding/videos/agent-welcome.md` (new)
- `content/onboarding/videos/tenant-welcome.md` (new)

## Test plan
**Manual review:**
- Script reads naturally aloud within duration
- Each scene has a single clear message
- First-time agent would understand without prior context

### 2026-04-23 — rentals-reviewer — changes requested

Good scripts overall: structure, shot list, accessibility checklist, SA-English, and brand voice are on target. Two issues must be fixed before this can go to testing.

1. **PLANNED features implied as available in voice-over (violates acceptance criterion "no audio-only information" AND brand voice rule "Honest about status").**
   The visual in agent Scene 3 correctly greys out PLANNED stages with a "Coming Soon" label, but the voice-over for that scene says *"from signing a mandate with the owner, through finding and **screening** tenants, to lease signing, rent collection, maintenance, and renewal."* `tenant_screening` is PLANNED (`content/product/features.yaml` line ~401-417). A deaf/HOH viewer reading only the caption will see the greyed marker; a sighted audio-only viewer hears screening presented as part of the live cycle. Same problem in agent Scene 5 VO: *"you can sign the mandate with the owner, **list it**, and book viewings"* — `property_advertising` is PLANNED. Fix: rewrite both VO lines (and their captions) to either omit PLANNED stages entirely, or add an explicit audio qualifier (e.g. "…screening and deposits are coming soon…"). The caption must match word-for-word.

2. **Tenant script voice-over is over calm-pace budget in two scenes.** Direction calls for "calm, reassuring" delivery, which is ~140–150 wpm. At those rates:
   - Scene 2 (14 s): 50 words ≈ 214 wpm — too fast.
   - Scene 5 (12 s): 41 words ≈ 205 wpm — too fast.
   Either trim 10–12 words per scene, or extend each by ~4 s. Agent Scene 5 (50 words in 16 s ≈ 188 wpm) is also tight — worth a trim pass.

Minor (not blocking, but worth addressing if you're in the file):
- Agent Scene 6 "Klikk wordmark, then three short bullet lines" — wordmark reveal + 3 staggered bullets + 4-s badge hold may overrun the 20 s budget; consider dropping the hold to 3 s to match the tenant outro.
- Tenant Scene 4 describes two chat turns — visual says "Camera sits on the chat for 5 seconds to let both exchanges land" but Scene is only 12 s total (0:36–0:48). Tight; verify in animatic.

No scope expansion needed; both fixes are edits to the two existing files.
