### 2026-04-22 — rentals-implementer

**What was done:**
Static code audit of the admin SPA and Astro marketing website. 10 findings identified (F-001 through F-010). All critical and serious structural violations were fixed in this commit. Audit document written at `docs/qa/accessibility-2026-04.md`.

**Fixes applied:**

1. `BaseModal.vue` — Added `role="dialog"`, `aria-modal="true"`, `aria-labelledby` (using Vue `useId()`). Close button gets `aria-label="Close dialog"`.
2. `BaseDrawer.vue` — Same ARIA pattern. Close button gets `aria-label="Close panel"`.
3. `SearchInput.vue` — `type="search"`, `:aria-label="ariaLabel || placeholder"`, new `ariaLabel` prop, clear button `aria-label="Clear search"`, search icon `aria-hidden`.
4. `ForgotPasswordView.vue` — Label `for`/input `id` association added. Error div gets `role="alert"`. `autocomplete="email"` added.
5. `RegisterView.vue` — All 7 label/input pairs now associated. Password show/hide button gets `:aria-label` + `aria-pressed`. Error div gets `role="alert"`. `autocomplete` attributes added throughout.
6. `klikk-components.css` — Global `:focus-visible { outline: 2px solid #2B2D6E; outline-offset: 2px; }` added (was missing for all non-agent-shell views).
7. `website/src/components/Header.astro` — `<nav aria-label="Main navigation">`, logo `aria-label="Klikk home"`, decorative period `aria-hidden`, `rel="noopener noreferrer"` on external login link.
8. `website/src/components/Footer.astro` — Footer links wrapped in `<nav aria-label="Footer navigation">`. SA flag emoji `aria-hidden`.
9. `website/src/styles/global.css` — `:focus-visible` ring added. Lighter version for dark nav/footer backgrounds.
