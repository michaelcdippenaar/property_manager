## Goal
Harden rent payment reconciliation for launch: correct matching of payments to tenants via unique reference, arrears flagging accuracy, receipt generation, partial payments.

## Acceptance criteria
- [x] Payment with correct unique reference auto-matches to tenant + lease
- [x] Payment with wrong reference flagged for manual reconciliation
- [x] Partial payments tracked, don't falsely mark rent paid
- [x] Arrears correctly flagged when rent overdue by N days (configurable)
- [x] Receipts generate on full payment
- [x] Owner dashboard reflects payment state in real time
- [x] No off-by-one errors around month boundaries or leap days

## Files likely touched
- `backend/apps/leases/models.py` (payment models)
- `backend/apps/leases/` (payment-related views)
- Owner dashboard views

## Handoff notes

### 2026-04-23 — implementer

**What was done**

Three new functions added to `backend/apps/payments/reconciliation.py`:

- `ingest_bank_payment(amount, payment_date, reference, *, payer_name, source, actor)` — reference-based auto-match against active leases via `Lease.payment_reference` (case-insensitive, stripped). Returns `(RentPayment, None)` on a single clean match, `(None, UnmatchedPayment)` for no-match, ambiguous match, empty reference, or matched-but-no-open-invoice. All paths write a `PaymentAuditLog` row.

- `flag_arrears(as_of_date, *, grace_days)` — returns a QuerySet of `unpaid`/`partially_paid` invoices whose `due_date < (as_of_date - grace_days)`. `grace_days` falls back to `settings.RENT_ARREARS_GRACE_DAYS` (default 3). Pure read — no mutations.

- `generate_receipt(invoice)` — returns a receipt dict for `paid`/`overpaid` invoices. Raises `ValueError` for any other status. Writes a `receipt_generated` audit log entry.

**Test coverage**

`backend/apps/payments/tests/test_rnt005_reconciliation_edges.py` — 46 new tests covering all 7 ACs:
- AC-1/AC-2: `TestReferenceAutoMatch` (5 tests) + `TestWrongReferenceUnmatched` (6 tests)

**Caveats**
- AC-6 ("real time") is validated at the serializer/ORM layer — the Django ORM always reads fresh from DB after `refresh_from_db()`. True WebSocket push to the owner dashboard (if ever added) would be a separate task.
- `ingest_bank_payment` picks the *oldest* open invoice when multiple unpaid periods exist on the same lease. If business logic should prefer the *most recent* period, that decision belongs to the PM.
