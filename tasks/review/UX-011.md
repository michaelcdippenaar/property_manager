## Goal
Return the tenant to the app with a confirmation state after they sign their lease in the external signing tab, so they know the action completed successfully.

## Acceptance criteria
- [x] After signing, the external signing flow redirects back to the tenant portal with a detectable query param (e.g. `?signed=1`) or the backend webhook triggers an in-app state update
- [x] On return, the pending-signing CTA on the Home dashboard is dismissed without requiring a manual page reload
- [x] Tenant sees an in-app confirmation: "Lease signed — awaiting countersignature from your agent"
- [x] If redirect-back is not feasible for web, a polling loop (max 5s interval) fires after the signing tab is opened and clears the CTA on confirmation
- [x] On native (Capacitor Browser), the `browserFinished` event is used to trigger the same state refresh
- [x] No regression: agents still receive the signing completion webhook as before

## Files likely touched
- `tenant/src/views/esigning/LeaseSigningView.vue` — lines 96-107, open signing URL + add callback/poll
- `tenant/src/views/home/HomeView.vue` — line ~152, `signingCta` flag refresh logic

2026-04-23 implementer: All acceptance criteria implemented. Three layers of detection:

1. **`PublicSignView.vue` (admin SPA)** — after `submitSignatures()` succeeds: posts `{ type: 'klikk_lease_signed' }` to `window.opener`; if a valid `?returnUrl=` query param was passed (origin-validated: same host, localhost, or *.tremly.co.za), redirects to `returnUrl?signed=1` after a 2.5s confirmation delay. Backend untouched — agent webhooks are unaffected.

2. **New composable `tenant/src/composables/useSigningStatus.ts`** — `startSigningWatch(leaseId, email)` runs two parallel mechanisms: (a) `window.addEventListener('message', ...)` catches the `postMessage` from the signing tab; (b) `setInterval` polls `/esigning/submissions/?lease=ID` every 5 seconds, max 24 polls (2 minutes). On native Capacitor, uses `Browser.addListener('browserFinished', ...)` to fire a single status check when the in-app browser closes. `stopSigningWatch()` cleans up all listeners.

3. **`LeaseSigningView.vue`** — `openSigning()` now appends `?returnUrl=<current page origin+path>` to the signing URL before opening, then calls `startSigningWatch`. When `signedAt` ref fires (from the composable), calls `refreshSubmissions()` to update signer statuses in-place (dismissing the "Sign Now" button), and sets `showSignedBanner = true` to display "Lease signed — awaiting countersignature from your agent". Also handles `?signed=1` on `onMounted` (for the redirect-back case). `stopSigningWatch` called in `onUnmounted`.

4. **`HomeView.vue`** — detects `?signed=1` on mount; sets `signingCta = false` and `showSignedConfirmation = true`, then calls `router.replace` to strip the query param. Added animated confirmation banner component.

**Caveats for reviewer:**
- The `returnUrl` validation whitelist covers `localhost` and `*.tremly.co.za`; if the tenant app ever runs on a different domain, the settings enum in `PublicSignView.vue` needs updating.
- The 2.5s redirect delay lets the tenant read the "Signed Successfully" screen before being whisked away. This is intentional UX — reviewers can adjust the delay if needed.
- On web, `window.opener.postMessage` only works within the same browser session (not when the tenant opens the link in a different browser). The polling fallback covers this case.
- `backend/apps/esigning/views.py` was not modified — the `returnUrl` approach is purely frontend.

2026-04-23 reviewer: Review requested changes. Blocking on one security issue and two small hardening items.

**Must fix (blocking):**

1. **`tenant/src/composables/useSigningStatus.ts` — add `event.origin` check in `onMessage`.** Currently:
   ```ts
   function onMessage(event: MessageEvent) {
     if (event.data?.type === 'klikk_lease_signed') { _markSigned() }
   }
   ```
   Any page the tenant visits that opens the tenant app via `window.open` (or embeds it) can post `{type: 'klikk_lease_signed'}` and spoof the signed state in the UI. Impact is UI-only (polling still fetches real backend state), but it flips `watchingSubId`, shows a false "Lease signed" banner, and dismisses the CTA until the next backend poll corrects it. The PM explicitly called this out.
   Fix: validate `event.origin` against the admin signing origin(s) — same host, `localhost` (dev), and `*.tremly.co.za` (matching the sender-side whitelist in `PublicSignView.vue`). Drop messages from any other origin silently.

**Should fix (non-blocking, please address in same pass):**

2. **`tenant/src/views/esigning/LeaseSigningView.vue` (onMounted, line ~185)** — when `route.query.signed === '1'`, the param is not stripped from the URL. `HomeView.vue` correctly calls `router.replace({ query: { ...route.query, signed: undefined } })`. Mirror that here so the param doesn't linger in history/analytics if the user navigates within the tenant app.

3. **`tenant/src/composables/useSigningStatus.ts` — `Browser.removeAllListeners()`** nukes all Capacitor Browser listeners globally. Prefer `const handle = await Browser.addListener(...)` then `handle.remove()` on stop, so unrelated listeners elsewhere in the tenant app aren't collateral-damaged.

**Verified as PM requested:**
- Origin whitelist in `PublicSignView.vue` correctly uses `URL.hostname` + `host.endsWith('.tremly.co.za')`; `attacker.tremly.co.za.evil.com` parses to hostname ending `.evil.com` — does NOT match. `evil-tremly.co.za` ends with `-tremly.co.za` (no leading dot) — does NOT match. Safe against the glob-style bypass.
- `HomeView.vue` correctly strips `?signed=1` via `router.replace` (vue-router 4 drops `undefined` query values).
- Polling: `_markSigned` calls `stopSigningWatch` → clears interval; `onUnmounted` also stops; `POLL_MAX_ATTEMPTS` (24 × 5s = 2 min) hard cap. No leaks under normal flow.
- Backend: diff confirms no changes under `backend/` — agent webhook path untouched.
