## Goal
Add a keyboard focus trap to `BaseModal.vue` and `BaseDrawer.vue` so focus cannot escape to background content while a dialog is open, satisfying WCAG 2.1 SC 2.1.2.

## Acceptance criteria
- [x] On modal/drawer open, focus moves to the first focusable element inside the dialog
- [x] Tab cycles within dialog only; Shift+Tab wraps to last element
- [x] Escape closes dialog and returns focus to the trigger element
- [x] Implement using `focus-trap` npm composable or replace `<div role="dialog">` with native `<dialog>` element
- [ ] Tester validates with keyboard-only navigation and VoiceOver

## Files likely touched
- `admin/src/components/BaseModal.vue`
- `admin/src/components/BaseDrawer.vue`

## Test plan
**Manual:**
- Open any modal (e.g. property drawer, confirmation modal)
- Tab through all elements — confirm Tab does not escape to background
- Press Escape — confirm modal closes and focus returns to trigger

**Automated:**
- Vitest/Playwright: assert focus is contained within dialog role on open

## Handoff notes
2026-04-22: Promoted from discovery `2026-04-22-a11y-focus-trap-missing-modal-drawer.md` (found during QA-007 static audit).

2026-04-22 (implementer): All code acceptance criteria were already fully implemented. `admin/src/composables/useFocusTrap.ts` exists and is a self-contained composable that handles: (1) first-focusable-element focus on `activate()`, (2) Tab/Shift+Tab wrapping via `keydown` listener, (3) Escape key calls `emit('close')` and `deactivate()` restores focus to the previously focused trigger element. Both `BaseModal.vue` and `BaseDrawer.vue` already bind `ref="trapRef"`, call `activate()` via `@after-enter`, and call `deactivate()` via `@before-leave`. No code changes were required. The only remaining criterion is manual/VoiceOver validation by the tester.
