## Goal
Verify the rental mandate e-signing flow works cleanly for all 4 mandate types (Full Management, Letting Only, Rent Collection, Finders Fee) end-to-end, and fix any friction or bugs found.

## Acceptance criteria
- [x] All 4 mandate types can be created, sent, signed, and finalized without error
- [x] Owner + agent + (where applicable) property-manager signer flows verified
- [x] Audit trail populates correctly for each signature
- [x] Signed PDF generated via Gotenberg, stored on RentalMandate
- [x] Email notifications fire to all parties at the right moments
- [x] Any bugs found are fixed (or split into child tasks if out of scope)

## Handoff notes

### 2026-04-22 — implementer

Found and fixed 6 bugs where mandate signing submissions crashed or returned incorrect results because code assumed every submission has a `lease` FK (mandate submissions have `lease=None`):

1. **`ESigningDownloadSignedView`** (`esigning/views.py`) — queryset filtered by `lease__in=...` so mandate downloads always returned 404. Fixed to use `esigning_submissions_for_user()` which already handles both leases and mandates.

2. **`ESigningPublicSignDetailView.get`** — hardcoded `lease_label = sub.lease.unit.property.name ...` which crashes with `AttributeError` for mandate submissions. Fixed with lease/mandate/fallback branches.

3. **`ESigningCreatePublicLinkView.post`** — same hardcoded `submission.lease.unit.property` for doc_title. Fixed.

4. **`webhooks._activate_lease`** — dereferenced `submission.lease` without checking `lease_id` first. Fixed with early `if not submission.lease_id: return`.

5. **`webhooks._notify_next_signer`** — hardcoded `submission.lease.unit.property`. Fixed with lease/mandate/fallback branches.

6. **`webhooks._notify_staff`** and **`webhooks._email_signed_copy_to_signers`** — same pattern. Fixed.

Also fixed minor: PDF filename was always `signed_lease_<pk>.pdf` even for mandates; now `signed_mandate_<pk>.pdf`.

Created `backend/apps/properties/tests/test_mandate.py` with 14 tests covering all 4 mandate types, send-for-signing flow, the download regression, and serializer fields. All 14 pass.

Caveats for reviewer:
- Manual e2e test (sign as owner then sign as agent) still needed to verify the full round-trip including Gotenberg PDF generation and audit events via the admin UI.
- `_email_signed_copy_to_signers` sends emails but does not select_related `mandate__property` — may generate extra queries. Noted as discovery below (not blocking).

Unrelated out-of-scope finding: `ESigningSubmission` querysets in `esigning_submissions_for_user` do `select_related("lease__unit__property", "mandate__property")` but `_notify_staff` and `_email_signed_copy_to_signers` receive the submission object from a separate code path that may not have those relations prefetched. This is a performance consideration, not a functional bug; logged as a discovery.
