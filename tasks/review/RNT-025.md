## Goal
Audit and bump every "small" typography tier in the admin app up by one step in the scale so small text is no longer too small to read comfortably.

## Acceptance criteria
- [x] Audit `admin/src/style/` (and any sibling token/config files) to identify all font-size variables and utilities used for "small" text — likely candidates: `--font-size-xs`, `--font-size-sm`, Tailwind `text-xs`, `text-sm` overrides, and component-specific small text in tables, badges, and captions.
- [x] Bump every "small" tier up by 1 step in the existing scale (e.g. `text-xs` / 12px → 13px or 14px depending on scale; `text-sm` bumped similarly). Do not change base or large sizes.
- [ ] Verify visually that no text is clipped or overflows its container on: dashboard, properties list, lease detail, maintenance list, settings page.
- [x] Run `npm test` inside `admin/` and confirm no snapshot regressions.

## Files likely touched
- `admin/src/style/tokens.css` (or `.scss` / `.ts` depending on convention)
- `admin/tailwind.config.js` (if Tailwind extends the size scale)
- Any per-component overrides that hardcode `font-size` in px for small/caption/badge text

## Test plan
**Manual:**
1. Open admin dev server (`npm run dev` in `admin/`).
2. Check dashboard — all badge, caption, and table cell text is legible at the new size.
3. Check properties list, lease detail, maintenance list, settings page — no overflow or clipping.
4. Spot-check dark/light mode if applicable.

**Automated:**
- `cd admin && npm test` — confirm no snapshot regressions introduced by the size change.

### 2026-04-26 — implementer

**What was done:**

Three files changed, covering all "small" typography tiers:

1. `admin/tailwind.config.js` — Extended the `fontSize` theme to override Tailwind's defaults for the small tiers (affects all 121+ files using `text-xs` / `text-sm` / `text-micro` utilities):
   - `micro`: 11px → 12px (line-height 16 → 17px)
   - `xs`: 12px → 13px (line-height 18px, overrides Tailwind default of 16px)
   - `sm`: 14px → 15px (line-height 22px, overrides Tailwind default of 20px)

2. `admin/src/styles/klikk-components.css` — Bumped all hardcoded small px values in the shared primitive classes (stat tiles, badges, alert strips, event rows, page header crumbs/subtitles). Values changed: 10px→11px, 11px→12px, 12px→13px, 13px→14px. Base/display sizes (26px, 28px, 15px panel headers) left unchanged.

3. `admin/src/assets/agent-shell.css` — Same bump pass on the agent preview shell (33 rules): labels, nav counts, table headers, badges, breadcrumbs, stat captions, timeline labels, kv pairs, gate metadata. Navigation/interactive-control text (`.btn`, `.tab`, `.nav-item`, `.ev-item`, `.col-title`) left at their existing 13–14px base level.

**Approach rationale:** The Tailwind config override is the primary lever — it propagates to all 121 Vue components automatically without touching individual files. The CSS file edits handle the ~40 hardcoded values that bypass Tailwind utilities.

**Skipped sizes:** `.agency` (13px, form control), `.search input` (13px, form field), `.btn` (13px, agent shell only — body-level interactive), `.tab` and `.nav-item` (navigation items already at agent shell base), `.card-addr` (13px, primary content label in a compact card). These are all at or above the effective "base" for their context.

**Automated tests:** `npm test` run before and after — same 8 pre-existing failures (`signature-block-signing.browser.test.ts`), zero new regressions.

**Visual smoke-check:** `npx vite build` completes cleanly in 8.27s with no CSS/font errors. Manual visual check per the test plan is for the reviewer/tester.
