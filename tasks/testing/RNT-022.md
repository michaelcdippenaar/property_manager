## Goal
Implement the Django backend for the AI Guide widget so the admin frontend can replace its in-process mock intent mapper with real Anthropic claude-3-5-haiku tool-use calls, with persistence and role-scoped action allowlisting.

2026-04-23 (reviewer): **Review requested changes.** Four blockers — all security.

1. **Portal is never validated against the caller's role (AuthZ fall-through).** `guide_views.py:174-175` silently coerces any unknown `portal` value to `"agent"`. A `TENANT` or `OWNER` user can send `portal: "agent"` (or `portal: "tenant"`, which falls through to agent) and receive the full agent tool allowlist — including `create_property`, `list_tenants`, `list_payments`. The server must derive the portal from `request.user.role` OR verify the submitted portal matches the user's role, and return **403** on mismatch (not coerce). This is the reviewer's explicit focus item: *"a tenant calling with an agent tool must get 403 not fall through"*. Add a test `test_tenant_cannot_use_agent_tools` and `test_owner_portal_mismatch_returns_403`.

2. **Unbounded `message` input — POPIA s19 integrity + cost / DoS.** `guide_views.py:165` accepts `request.data.get("message")` with no length cap. A caller can submit megabytes of text that get forwarded to Anthropic (cost) and persisted verbatim to `GuideInteraction.message` (a `TextField` with no limit in `models.py:125`). Fix: enforce a server-side cap (e.g. 1000 chars) in the view and reject >cap with 400. Document retention of `GuideInteraction.message` (or truncate) per POPIA s14/s19.

3. **Dead `.env` fallback for `ANTHROPIC_API_KEY`.** `guide_views.py:51-67` opens `backend/.env` directly at request time and parses it by hand — despite `settings.ANTHROPIC_API_KEY` already resolving via `_env_data` (base.py:222). Remove the file-read fallback entirely; rely on `settings.ANTHROPIC_API_KEY`. Reading arbitrary dotfiles from view code is an anti-pattern and masks misconfiguration.

4. **Rate-limit scope is cosmetic.** `AIGuideThrottle` sets both `rate = "20/min"` and `scope = "ai_guide"`. With a hard-coded `rate`, DRF ignores `DEFAULT_THROTTLE_RATES[scope]`, so the scope label does nothing. Either drop the `scope` attribute, or move the rate into `REST_FRAMEWORK["DEFAULT_THROTTLE_RATES"]["ai_guide"] = "20/min"` and remove the hard-coded `rate`. Minor, but do it while you're in there.

2026-04-23 (implementer — blocker fixes):

**All four blockers resolved. 12/12 tests passing.**

**Fix 1 — Privilege escalation (portal spoofing):** Removed client-supplied `portal` field entirely. Added `_ROLE_TO_PORTAL` dict mapping every `User.Role` value to a portal string. `_portal_for_user()` looks up the mapping; any role not in the map (TENANT, VIEWER, ACCOUNTANT) returns `None` and the view returns 403. Deleted `VALID_PORTALS` constant (no longer needed). Tests added: `test_tenant_cannot_use_ai_guide_returns_403`, `test_tenant_portal_field_ignored_still_403`, `test_agent_portal_derived_from_role` (verifies payload `portal=owner` is ignored for an ESTATE_AGENT user, and the persisted interaction has `portal=agent`).

**Fix 2 — Unbounded message (POPIA/cost):** Added `MESSAGE_MAX_LENGTH = 2000` constant and a guard in `post()` that returns 400 with a descriptive message before the Anthropic call or DB persist. Tests added: `test_oversized_message_returns_400`, `test_message_at_max_length_is_allowed`.

**Fix 3 — Dead .env fallback:** Removed `_get_anthropic_api_key()` function entirely (including the `os` import). `_call_guide()` now reads `getattr(settings, "ANTHROPIC_API_KEY", "") or ""` directly, consistent with how `base.py` already resolves the key. Existing tests that patched `_get_anthropic_api_key` updated to patch `settings.ANTHROPIC_API_KEY` via the pytest `settings` fixture instead.

**Fix 4 — Throttle scope is cosmetic:** Removed `scope = "ai_guide"` from `AIGuideThrottle`. The hardcoded `rate = "20/min"` is the sole source of truth. Test added: `test_rate_limit_triggers_429` — patches rate to `1/min`, fires two requests, asserts second gets 429.

**What I verified and was happy with:**
- `is_tool_allowed` double-check runs **after** the model returns (`guide_views.py:108`) — catches hallucinated tools. Correct.
- `IsAuthenticated` permission + 401 test present. Correct.
