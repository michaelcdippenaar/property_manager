## Goal
Prevent `{% if cta_url %}` and `{% if note %}` blocks from being silently overwritten when `base.mjml` is recompiled, by automating the compile + patch step and adding a CI guard.

## Acceptance criteria
- [x] `scripts/compile_email_templates.sh` (or a `Makefile` target) runs `npx mjml base.mjml -o compiled/base.html` and then applies the Django `{% if %}` patch blocks via `sed` or a Python post-processing script
- [x] A CI check (pre-commit hook or GitHub Actions step) fails if `compiled/base.html` is out of sync with the current `base.mjml` or if the conditional blocks are absent
- [x] Compile step documented in `backend/apps/notifications/NOTIFICATIONS.md`
- [x] Manually running the compile script produces a `compiled/base.html` with the correct `{% if cta_url %}` and `{% if note %}` blocks intact

## Files likely touched
- `scripts/compile_email_templates.sh` (new)
- `Makefile` (new target `compile-emails`)
- `backend/apps/notifications/NOTIFICATIONS.md`
- `.github/workflows/ci.yml` (optional CI check)
- `.pre-commit-config.yaml` (optional pre-commit hook)

## Test plan
**Manual:**
- Edit `base.mjml` (minor whitespace change); run compile script; confirm conditional blocks are present in `compiled/base.html`
- Manually remove `{% if cta_url %}` from `compiled/base.html`; run CI check — it should fail

**Automated:**
- CI email-template check step exits non-zero if compiled output is stale or missing conditionals

Promoted from discovery: `2026-04-22-mjml-build-script.md` (OPS-005). Without this, any `base.mjml` edit silently breaks email conditional rendering with no error.

**2026-04-22 — implementer**
Delivered all four acceptance criteria:

- `scripts/compile_email_templates.sh` — two-phase script: (1) strips standalone `{% if %}`/`{% endif %}` wrapper lines (regex anchored to whole-line matches so inline conditionals like `{% if unsubscribe_url %}` inside `<mj-text>` content survive), (2) runs `npx mjml`, (3) calls `scripts/patch_email_conditionals.py` to re-insert the CTA and note wrappers. Supports `--check` mode for CI.
- `scripts/patch_email_conditionals.py` — idempotent Python post-processor that uses `<!-- BLOCK: optional CTA button -->` and `<!-- BLOCK: optional secondary note -->` HTML comment anchors (MJML preserves these) to insert `{% if cta_url %}` / `{% if note %}` wrappers. Raises a `RuntimeError` with a clear message if the anchor comments are missing.
- `Makefile` added with `compile-emails` and `check-emails` targets.
- `.github/workflows/ci.yml` — new `email-templates` job runs `make check-emails` (Node 22, no DB needed).
- `.pre-commit-config.yaml` — local hook runs `compile_email_templates.sh --check` when any file under `email_templates/` is staged.
- `NOTIFICATIONS.md` updated with full compile-step docs and instructions for adding new conditional blocks.

Caveat: MJML 5 strips `{% if %}` / `{% endif %}` when they appear as standalone lines wrapping `<mj-button>` / `<mj-text>` components, but passes through inline Django syntax inside text content. The script handles both cases correctly. Smoke-checked: `make compile-emails` and `make check-emails` both pass.
