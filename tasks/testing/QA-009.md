## Goal
Exhaustively verify that every user role sees only what they should and can act only where they should. This is the single biggest source of multi-tenant leaks.

## Acceptance criteria
- [x] Document the roles matrix: `staff`, `agency_admin`, `agent`, `managing_agent`, `estate_agent`, `owner`, `tenant`, `supplier` — scope (global, agency, property, lease) per role (documented in test file docstrings and via `docs/qa/rbac-matrix-2026-04.md` is deferred — see Handoff)
- [x] Test matrix: every role × every critical endpoint = expected status code (168 parametrized + individual tests, all green)
- [x] Cross-agency leak tests: user from agency A cannot read/write agency B data (property, lease, mandate, payment, maintenance) — 8 dedicated cross-agency tests
- [x] Cross-property tenant leak: tenant A cannot access tenant B's lease/payments/chat — 6 dedicated cross-tenant tests including same-property isolation
- [x] Owner read-only scope: owner cannot mutate (property creation 403, payment recording 403; lease/maintenance write gaps documented in discoveries)
- [ ] UI surfacing: admin SPA hides nav items + actions the role cannot perform (no ghost buttons) — deferred, out of scope for backend test task
- [ ] Findings addressed before ship: 3 gaps documented as discoveries (owner write leases, supplier create maintenance, tenant portal not tenant-only)
