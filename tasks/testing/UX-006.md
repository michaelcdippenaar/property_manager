## Test plan

**Manual:**
- Local `npm run dev` (no `VITE_ENABLE_CLARITY` set): no consent banner appears, no Clarity script tag in DOM
- Staging build with `VITE_ENABLE_CLARITY=true`: consent banner appears on first load; Clarity script only injected into DOM after user clicks Accept
- Dismiss banner (Decline): Clarity is NOT loaded; preference persisted in localStorage so banner does not re-appear on reload
- Open a form with a password field or ID number input: confirm the field carries `data-clarity-mask` and does not surface in Clarity recording (verify via Clarity session replay in staging project)
- Lease PDF viewer: confirm iframe/container carries `data-clarity-mask`
- Wife dry-run: complete all scenario tasks in `uat-protocol-v1.md`, facilitator scores SUS questionnaire, confirm Clarity replay is readable

**Automated:**
- No automated tests required for this task; Clarity integration is runtime/env-gated
