### 2026-04-22 — reviewer: changes requested

Two fixes required before this can move to testing.

**1. `website/src/components/ConsentBanner.astro`, lines 9–10 — incorrect JSDoc cookie value documentation**

The frontmatter comment reads:
```
 * Values: "essential" (accepted) | "declined"
```
but the actual cookie values set by the script are `'analytics'` (accept button) and `'essential'` (decline button). The comment is wrong on both counts. When analytics is wired up (e.g. Plausible script gating on `klikk_consent`), this comment will actively mislead the implementer into using the wrong value check. Fix: change the comment to:
```
 * Values: "analytics" (accepted analytics) | "essential" (essential only)
```

**2. `website/src/layouts/BaseLayout.astro`, line 114 — `"telephone": null` in LocalBusiness JSON-LD**

`null` is not a valid Schema.org value and will cause a Google Rich Results Test warning (and potentially a structured-data validation error in Search Console). Remove the `telephone` key entirely rather than emitting `null`:
```json
// Remove this line:
"telephone": null,
```
If a phone number is added later, restore the field with an actual value.

No security issues found. Lighthouse and mobile criteria are correctly deferred to the tester — the two items above are the only blockers.
