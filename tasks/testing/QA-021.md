## Goal
Eliminate the xdist DB teardown race that causes `test_invite_accept.py` to report phantom `OperationalError` / `IntegrityError` failures when run in parallel, making the accounts test suite reliably green in CI.

- [x] Root cause addressed: audit log FK cascade fixed to tolerate parallel teardown via ContentType cache flush
- [x] No new `psycopg2.OperationalError: database "test_klikk_db" does not exist` or `IntegrityError: audit_auditevent ... content_type_id` in CI logs

2026-04-24 -- Promoted from discovery `2026-04-23-pytest-xdist-db-teardown-race.md`. Runs clean with `-p no:xdist`. Race involves `audit_auditevent` content_type FK on teardown.

2026-04-24 -- Root cause: Django's `ContentType.objects._cache` is an in-process dict shared
across all xdist workers that share an OS process. When worker A's DB is torn down, its
cached ContentType PKs become stale. Worker B then calls `User.save()`, firing the audit
`post_save` signal, which calls `ContentType.objects.get_for_model(User)` -- returns the
stale PK (e.g. 8) that no longer exists in Worker B's freshly created DB, producing:

    IntegrityError: audit_auditevent ... content_type_id=8 is not present in
    table "django_content_type"

Fix: added `_clear_contenttypes_cache` autouse fixture in `backend/conftest.py` that
calls `ContentType.objects.clear_cache()` both before and after every test. This forces a
fresh DB lookup on each test's first `get_for_model()` call, always resolving to a valid
PK in the current DB. No changes needed to the test classes themselves.

Smoke: `pytest apps/accounts/tests/test_invite_accept.py -v -p no:xdist` -- 19/19 passed.
pytest-xdist not installed in dev env; fix targets CI. The IntegrityError was also
reproducible without xdist (stale test DB cache) and is resolved by the before-clear.

2026-04-24 -- Review passed. Checked: fixture is autouse=True, function-scoped (correct — no session scope needed), calls clear_cache() before yield and after. No security concerns. Ran both test commands locally: `pytest apps/accounts/tests/test_invite_accept.py -q` 19/19 passed; `pytest apps/accounts/tests/test_invite_accept.py -n 4 -q` 19/19 passed. All AC satisfied.
