## Goal
Add the three RHA s5(3) mandatory-term fields to the `Lease` model and wire them into `rha_check.py` so the compliance gate can flag leases that are missing escalation clause, renewal clause, or domicilium address before signing.

## Acceptance criteria
- [x] `Lease.escalation_clause` (TextField, blank=True) added and migrated
- [x] `Lease.renewal_clause` (TextField, blank=True) added and migrated
- [x] `Lease.domicilium_address` (TextField, blank=True) added and migrated
- [x] Migration created and does not break existing lease records or the occupancy signal (`apps/leases/signals.py`)
- [x] `rha_check._check_mandatory_terms` raises blocking flags (`MISSING_ESCALATION_CLAUSE`, `MISSING_RENEWAL_CLAUSE`, `MISSING_DOMICILIUM`) when the respective fields are empty
- [x] Lease serializer and admin API expose all three new fields for read/write
- [x] Tests cover: (a) all three blocking codes fire on an empty lease, (b) no flags fire when all three are populated, (c) migration smoke-test on a pre-existing lease record

## Files likely touched
- `backend/apps/leases/models.py`
- `backend/apps/leases/migrations/` (new migration)
- `backend/apps/leases/rha_check.py`
- `backend/apps/leases/serializers.py`
- `backend/apps/leases/tests/test_rha_gate.py`

## Test plan
**Manual:**
- Create a new lease without escalation/renewal/domicilium fields set; attempt e-sign — RHA gate must block with the three new codes
- Populate all three fields; attempt e-sign — gate must pass

**Automated:**
- `cd backend && pytest apps/leases/tests/test_rha_gate.py -v`
